// Purge all kb_* keys from localStorage
function purgeUserState() {
  // Purge known keys + all kb_* prefixed keys
}

// Check if user changed and purge if needed
function checkUserChanged(newEmail: string): boolean {
  const lastEmail = localStorage.getItem(LAST_USER_KEY);
  if (lastEmail && lastEmail !== newEmail) {
    purgeUserState();
    return true;
  }
}

// Redirect to onboarding if no tenant (hard gate)
if (status === 'authenticated' && user && !user.tenantId && !isNoTenant(pathname)) {

## Clés localStorage Purgées

À chaque logout ou changement d'utilisateur:

| Clé | Description |
|-----|-------------|
| `kb_current_tenant` | Tenant actuel sélectionné |
| `kb_focus_mode` | État du mode focus |
| `kb_onboarding_state` | Progression onboarding |
| `kb_last_tenant` | Dernier tenant utilisé |
| `kb_stats_cache` | Cache des statistiques |
| `kb_conversations_cache` | Cache des conversations |
| `kb_orders_cache` | Cache des commandes |
| `kb_last_user_email` | Email du dernier utilisateur |
| `kb_*` | Toutes les clés préfixées kb_ |

## Flux Utilisateur Final

### Utilisateur AVEC tenants
```
Login → /select-tenant → Sélection tenant → /inbox
```

### Utilisateur SANS tenants
```
Login → /select-tenant → (0 tenants) → /onboarding → Création espace
```

### Logout
```
Logout → Purge kb_* localStorage → /login
```

### Switch User
```
Login user B → Détection changement email → Purge état user A → Chargement état user B
```

- Utilisateur authentifié sans tenant: redirigé vers /onboarding (pas de données démo)
