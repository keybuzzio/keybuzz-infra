## Cause racine

**Deux endpoints BFF différents avec des comportements différents :**

| Aspect | `/api/auth/tenants` (AVANT) | `/api/tenant-context/tenants` (APRÈS) |
|--------|---------------------------|--------------------------------------|
| Backend appelé | `/tenant-context/me` | `/tenant-context/tenants` |
| URL source | `getApiUrl()` (interne) | ~~`NEXT_PUBLIC_API_URL`~~ → `getApiUrl()` (interne) |
| Comportement | Retourne tenants + currentTenantId via `/me` | Retourne liste tenants directe |

Le problème : `/tenant-context/me` peut ne pas retourner les tenants si `currentTenantId` n'est pas encore défini (premier login, session pas propagée). Tandis que `/tenant-context/tenants` retourne **toujours** la liste des tenants pour un email donné.

**Problème secondaire** : `/api/tenant-context/tenants` utilisait `NEXT_PUBLIC_API_URL` (URL publique) au lieu de `getApiUrl()` (URL interne K8s). En cluster, l'URL publique peut être lente ou échouer.

**Problème tertiaire** : `/api/debug-spaces` et `/api/debug-profile-billing` appelaient `getServerSession()` sans `authOptions`, causant une résolution de session incomplète.

## Fix appliqué (4 fichiers)

### 1. `app/select-tenant/page.tsx`

```
AVANT: fetch('/api/auth/tenants')        → backend /tenant-context/me
APRÈS: fetch('/api/tenant-context/tenants') → backend /tenant-context/tenants
```

- Constante `TENANTS_ENDPOINT = '/api/tenant-context/tenants'` (source unique)
- Constante `SELECT_ENDPOINT = '/api/auth/select-tenant'` (séparation claire)
- Retry réduit à 2 (car l'endpoint est maintenant fiable)
- `cache: 'no-store'` ajouté pour éviter le cache Next.js

### 2. `app/api/tenant-context/tenants/route.ts`

```
AVANT: const API_URL = process.env.NEXT_PUBLIC_API_URL || '';
APRÈS: const API_URL = getApiUrl();  // API_URL_INTERNAL > API_URL > NEXT_PUBLIC_API_URL
```

- Import de `getApiUrl` depuis `@/src/lib/api-url`
- `cache: 'no-store'` ajouté
- Logging amélioré en cas d'erreur

### 3. `app/api/debug-spaces/route.ts`

```
AVANT: getServerSession()
APRÈS: getServerSession(authOptions)
```

### 4. `app/api/debug-profile-billing/route.ts`

```
AVANT: getServerSession()
APRÈS: getServerSession(authOptions)
```
