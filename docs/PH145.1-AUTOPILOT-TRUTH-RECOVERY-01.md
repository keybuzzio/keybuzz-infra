### UX attendue (safe_mode = true)

```
1. Message client entrant
2. Autopilot evaluate → LLM genere reponse (confiance >= 0.5)
3. Draft stocke dans ai_action_log (blocked_reason=DRAFT_GENERATED)
4. User ouvre conversation dans inbox
5. InboxTripane useEffect fetch GET /api/autopilot/draft
6. Si hasDraft=true → setAutopilotDraft + setAutopilotAutoOpen
7. Drawer AISuggestionSlideOver s'ouvre automatiquement
8. Affiche : texte draft + confiance + escalation warnings
9. 3 boutons : Valider et envoyer / Modifier / Ignorer
10. Chaque action appelle POST /api/autopilot/draft/consume
11. Draft disparait (plus de ghost drafts)
```

## 5. Cause racine

### Hypothese verifiee : "Le draft n'est pas visible dans le navigateur"

Si cette observation est rapportee, les causes possibles sont :

1. **Tenant de test** : ecomlg-001 est PRO. Le gating PH142-O1.2 indique "Brouillon IA: Non" pour PRO. Le draft est GENERE (cote API/engine) mais le CLIENT pourrait ne pas le montrer si le plan est PRO. Il faut tester avec un tenant AUTOPILOT (switaa-sasu-mnc1ouqu) pour confirmer.
2. **Pas de draft frais** : Le draft existant (10 avril) est pour une conversation specifique. Si l'utilisateur ouvre une AUTRE conversation, il n'y aura pas de draft.
3. **Session expiree** : Si le cookie NextAuth est expire, le BFF retourne 401 et le fetch echoue silencieusement.
4. **Le mode supervised de ecomlg-001** : En mode supervised avec PRO, `resolveIAMode` retourne `MODE_NOT_AUTOPILOT:suggestion`, ce qui empeche l'evaluation automatique de NOUVELLES conversations. Les drafts existants restent accessibles mais aucun NOUVEAU draft ne sera genere.

### Cause racine unique si "bug" rapporte

**Le brouillon IA est une feature AUTOPILOT uniquement** (confirme par PH142-O1.2 table de verite). Sur un tenant PRO comme ecomlg-001, l'engine genere des drafts (si invoque manuellement via /autopilot/evaluate) mais le plan PRO NE PERMET PAS le mode autopilot complet. Le comportement attendu pour PRO est "Aide IA" (suggestions manuelles), pas "Brouillon IA automatique".
