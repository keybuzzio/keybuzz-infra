## Problèmes identifiés et corrigés

### 1. Layout /signup avec sidebar (bug)

**Problème**: La page `/signup` affichait la sidebar complète (Inbox, Commandes, Fournisseurs, etc.) alors que `/login` était propre.

**Cause**: `ClientLayout.tsx` avait une liste `PUBLIC_ROUTES` qui n'incluait pas `/signup`.

**Fix**: 
```typescript
// AVANT
const PUBLIC_ROUTES = ["/login", "/logout", "/select-tenant"];

// APRÈS
const PUBLIC_ROUTES = ["/login", "/logout", "/select-tenant", "/signup", "/invite", "/auth", "/pricing", "/start"];
```

**Fichier**: `src/components/layout/ClientLayout.tsx`

---

### 2. Boutons SSO morts

**Problème**: Les boutons "Continuer avec Google" et "Continuer avec Microsoft" sur `/signup` ne faisaient rien au clic.

**Cause**: Les boutons n'avaient pas de handler `onClick`.

**Fix**: Ajout des handlers OAuth:
```typescript
import { signIn } from 'next-auth/react';

const handleGoogleLogin = () => {
  signIn('google', { callbackUrl: '/select-tenant' });
};

const handleMicrosoftLogin = () => {
  signIn('azure-ad', { callbackUrl: '/select-tenant' });
};
```

**Fichier**: `app/signup/page.tsx`

---

### 3. redirectTo non stable

**Problème**: Le `returnTo` cookie existait sur `/login` mais n'était pas lu après l'authentification réussie.

**Cause**: `onLoginSuccess` dans `AuthGuard.tsx` redirigeait toujours vers `/select-tenant` sans vérifier le cookie.

**Fix**:
```typescript
const onLoginSuccess = useCallback(() => {
  setStatus('loading');
  fetchAuth().then(() => {
    const returnTo = getReturnToCookie();
    if (returnTo) {
      clearReturnToCookie();
      window.location.href = returnTo;
    } else {
      router.replace('/select-tenant');
    }
  });
}, [fetchAuth, router]);
```

**Fichier**: `src/components/auth/AuthGuard.tsx`
