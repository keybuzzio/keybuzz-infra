# PH143-E.1 — Fix Aide IA Suggestion Drawer BFF Tenant Context

## 1. Résumé exécutif

Le bug bloquant "Impossible de générer une suggestion" dans le drawer Aide IA a été identifié, corrigé et validé en navigateur réel.

**Cause racine** : Le client `ai.service.ts` appelait `API_CONFIG.baseUrl + '/ai/assist'` (= `https://api-dev.keybuzz.io/ai/assist`) directement depuis le navigateur, **contournant la route BFF** `/api/ai/assist`. L'appel direct au backend manquait du header `X-User-Email` requis par le plugin `requestContext` de Fastify.

**Correction** : Deux fichiers modifiés pour router la suggestion IA via la BFF avec authentification et tenant context complets.

---

## 2. Cause racine exacte

### Flux AVANT (cassé)
```
Navigateur → POST https://api-dev.keybuzz.io/ai/assist
  └─ Body: { tenantId, conversationId, payload }
  └─ Headers: Content-Type: application/json (SEUL header)
  └─ ❌ Pas de X-User-Email → requestContext vide → tenantGuard reject
```

### Flux APRÈS (corrigé)
```
Navigateur → POST /api/ai/assist (BFF Next.js, même origine)
  └─ BFF extrait session NextAuth → userEmail
  └─ BFF extrait tenantId du body JSON
  └─ BFF → POST http://keybuzz-api.keybuzz-api-dev.svc.cluster.local:3001/ai/assist
     └─ Headers: X-User-Email, X-Tenant-Id, Content-Type
     └─ Body: forwarded as-is
     └─ ✅ requestContext complet → tenantGuard OK → suggestion générée
```

---

## 3. Fichiers modifiés

### `src/services/ai.service.ts` (client)
**Avant** : `assistAI()` passait par `fetchAI()` qui préfixait `API_CONFIG.baseUrl` (URL externe backend).
**Après** : `assistAI()` appelle directement `fetch('/api/ai/assist', ...)` (route BFF relative). La gestion d'erreurs budget/KBActions est préservée inline.

### `app/api/ai/assist/route.ts` (BFF)
**Avant** :
- `API_URL` = `NEXT_PUBLIC_API_URL` (URL externe, sous-optimal pour server-to-server)
- `tenantId` lu uniquement depuis le header `X-Tenant-Id` (jamais envoyé par le client)
- Pas de validation `tenantId` manquant

**Après** :
- `getApiUrl()` = `API_URL_INTERNAL || API_URL || NEXT_PUBLIC_API_URL` (K8s internal preferred)
- `tenantId` lu depuis header OU body JSON (`body.tenantId`)
- Validation 400 si `tenantId` absent
- `X-Tenant-Id` toujours envoyé au backend
