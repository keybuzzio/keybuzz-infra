## 3. Corrections appliquees

### 3.1 Encart compact panneau droit

Bloc IIFE insere entre le header "Commande" et le contenu scrollable :

- **Type badge** : Bell (Notification Amazon) / Package (Demande de retour) / Truck (Suivi de livraison)
- **Duree** : calcul jours depuis date commande
- **Chevron** : collapsible (ChevronUp/ChevronDown)
- **ACTIONS** : pills ambre detectees depuis les messages inbound (Remboursement initie, Retour autorise, Reclamation ouverte, Fournisseur contacte)
- **ETAT** : label statut conversation (En traitement, En attente, Resolu) + extras (Gere par IA, SAV actif/resolu)
- **Styling** : `rounded-lg border border-gray-200 bg-white shadow-sm` — bloc visuel distinct, PAS une simple ligne, PAS un border-b

**Condition d'affichage** : le bloc n'apparait QUE si :
- `isSystemNotification === true` OU
- `conversationType === "RETURN_REQUEST"` OU
- `subject` contient des mots-cles livraison OU
- `caseSummary` a des actions/extras

Si aucune condition n'est remplie → **rien n'est rendu** (pas de ligne parasite).

### 3.2 Messages Amazon auto — fond ambre

Classification conservative multi-signaux (`isAmazonAutoMessage`) :
- Requiert **au moins 2 signaux** parmi 10 patterns (copyright+amazon, seller central, preferences de notification, ne pas repondre, cet email a ete envoye, amazon EU S.a r.l., etc.)
- **Faux positifs proteges** : un message client mentionnant "Amazon" seul ne declenche PAS la classification (il faudrait "copyright" + "Amazon" OU "Seller Central" etc.)
- Styling : `bg-amber-50 border-amber-200` (ambre clair)
- Badge inline : Bell + "Amazon" (ambre)

### 3.3 Messages client — badge "Client"

- Tout message inbound NON-auto, NON-interne, NON-fournisseur recoit le badge "Client" (orange)
- Fond blanc inchange

### 3.4 Bouton "Ouvrir la commande"

Deja conforme (`bg-purple-600`) — aucune modification.

## 4. Donnees calculees (InboxTripane → OrderSidePanel)

| Prop | Source | Utilisation |
|---|---|---|
| `conversationType` | `selectedConversation.conversationType` | Determination type badge |
| `isSystemNotification` | `selectedConversation.isSystemNotification` | Determination type badge |
| `conversationSubject` | `selectedConversation.subject` | Detection livraison par mots-cles |
| `caseSummary` | `computeCaseSummary(selectedConversation)` | ACTIONS + ETAT |

`computeCaseSummary()` analyse les messages de la conversation :
- Recherche patterns refund/return/reclamation dans les messages inbound
- Detecte messages fournisseur (`messageSource.startsWith("SUPPLIER")`)
- Mappe le statut conversation vers un label francais
- Detecte `assignedType === "ai"` et `savStatus`
