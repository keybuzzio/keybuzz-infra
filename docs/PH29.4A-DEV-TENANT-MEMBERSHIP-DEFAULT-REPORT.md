```javascript

```
**Réponse**:

```json
    "email": "ludo.gonthier@gmail.com",
    "name": "Ludovic Gonthier"
  },
  "tenants": [{
    "id": "ludovic-ml5koskd",  // TENANT DIFFÉRENT!
    "name": "Ludovic",
    "role": "owner"
  }],
  "currentTenantId": "ludovic-ml5koskd"

```

### 4.2 Anomalie
| Source | user.id | currentTenantId |
|--------|---------|-----------------|
| Backend direct | `0156d1ac-...` | `ecomlg-001` ✅ |
| Pod client → backend | `0156d1ac-...` | `ecomlg-001` ✅ |
| Navigateur → backend | `0156d1ac-...` | `ecomlg-001` ✅ |
| Navigateur → `/api/tenant-context/me` | `43a1d34c-...` | `ludovic-ml5koskd` ❌ |

### 4.3 Cause racine suspectée
1. **Deux user IDs différents** pour le même email:
   - DB backend: `0156d1ac-0863-431f-9717-216af804e7ef`
   - API client: `43a1d34c-b8de-4226-b8db-0f4da87924a7`

2. **Origine du tenant `ludovic-ml5koskd`**:
   - Créé lors d'un onboarding précédent sur DEV après nettoyage
   - N'existe PAS dans la table `tenants` de la DB backend
   - N'existe PAS dans `user_tenants` pour cet utilisateur

3. **Source de la discordance**:
   - Le client Next.js semble utiliser une source de données différente
   - Possible cache CDN/proxy ou problème de session NextAuth
