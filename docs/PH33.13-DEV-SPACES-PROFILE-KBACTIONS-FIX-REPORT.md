## Résumé des modifications

### 1. Correction hardcodes (`useCurrentPlan.tsx`)

**Problème**: Valeurs hardcodées pour le plan par défaut (`'PRO'`), le cycle (`'monthly'`), les channels (`3`) et l'intervalle de polling (`500`).

**Fix**: Extraction en constantes nommées sourced depuis `planCapabilities.ts` :
- `DEFAULT_PLAN = 'STARTER'` (cohérent avec le fallback logique)
- `DEFAULT_CYCLE: BillingCycle = 'monthly'`
- `DEFAULT_CHANNELS = PLAN_CAPABILITIES.STARTER.maxChannels`
- `TENANT_ID_POLL_MS = 500`

### 2. Fix "Impossible de charger vos espaces" (`TenantProvider.tsx`)

**Cause racine**: Après un logout/login, le backend peut mettre un court délai à reconnaître la session. Si le premier appel à `/tenant-context/tenants` échoue (401 ou timeout), le provider affichait une erreur rouge permanente sans retry.

**Fix**:
- Ajout d'un mécanisme de retry (max 2 retries, délai 1.5s entre tentatives)
- Auto-sélection du tenant quand `currentTenantId` est vide et l'user n'a qu'un seul tenant
- L'erreur n'est affichée qu'après épuisement de tous les retries

### 3. KBActions dotation visible (`billing/plan/page.tsx`)

**Problème**: La dotation mensuelle KBActions n'était pas visible dans le header de la carte plan.

**Fix**: Ajout d'un bloc dédié dans le header gradient :
```
Dotation mensuelle : 2 000 KBActions/mois
```
Affiché uniquement si `capabilities.kbActionsMonthly > 0`.
Valeur lue dynamiquement depuis `PLAN_CAPABILITIES[plan].kbActionsMonthly` (source unique `planCapabilities.ts`).

### 4. Debug endpoints BFF (DEV ONLY)

Deux nouveaux endpoints créés pour diagnostic sans DevTools :

**`/api/debug-spaces`** (GET)
- Retourne : `sessionEmail`, statut backend `/tenant-context/tenants`, `/tenant-context/me`

**`/api/debug-profile-billing`** (GET, optionnel `?tenantId=...`)
- Retourne : `tenantId` résolu, profil entreprise, billing courant (plan, cycle, statut), dotation KBActions

### 5. Company profile

**Constat**: Le profil entreprise est déjà correctement persisté au moment du `create-signup` dans la table `tenant_metadata`. La page Settings le lit via `/api/tenant-context/profile/[tenantId]`. Chaîne fonctionnelle.
