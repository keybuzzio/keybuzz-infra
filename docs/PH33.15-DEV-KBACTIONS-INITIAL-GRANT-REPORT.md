# PH33.15 — Crédit initial KBActions à l'onboarding (DEV ONLY)

**Date** : 2026-02-16  
**Statut** : Déployé DEV, en attente validation  
**PROD** : Non touchée

---

## Cause racine (problème pré-existant)

1. **Case mismatch** : `tenants.plan` stocke `'PRO'` (majuscule), mais `PLAN_KBACTIONS_MONTHLY` avait des clés en minuscule (`'pro'`). `getPlanIncludedKBActions('PRO')` retournait **0** au lieu de **1000**.
2. **Aucun crédit initial** : Le webhook Stripe (`handleSubscriptionChange`) créait la `billing_subscriptions` mais ne créditait jamais de KBActions au wallet.
3. **Divergence config** : Backend avait `autopilote=3500`, client avait `AUTOPILOT=2000`. Aligné sur la spécification : **2000**.

---

## Implémentation

### 1) Fix `getPlanIncludedKBActions` (case-insensitive)

**Fichier** : `keybuzz-api/src/config/kbactions.ts`

- Ajout de `.toLowerCase()` dans `getPlanIncludedKBActions(plan)`
- Ajout de l'alias `'autopilot': 2000` (DB stocke `AUTOPILOT`, pas `AUTOPILOTE`)
- Alignement `autopilote`/`autopilot`/`business` → **2000** (conformément à la spec et au client)

### 2) Fonction `grantInitialKBActions` (idempotente)

**Fichier** : `keybuzz-api/src/services/ai-actions.service.ts`

```
grantInitialKBActions(tenantId, plan, subscriptionId)
```

- **Idempotency key** : `initial_grant_{subscriptionId}` (via `ai_actions_ledger.request_id`)
- **Vérifie** si une entrée `reason='initial_grant'` + `request_id=idempotencyKey` existe déjà
- Si oui → ne fait rien (retourne `alreadyGranted: true`)
- Si non → crédite `remaining + amount`, met à jour `included_monthly`, log dans ledger

**Ledger row** :
| Champ | Valeur |
|---|---|
| `tenant_id` | tenant-xxx |
| `delta` | +1000 (ou +2000 selon plan) |
| `reason` | `initial_grant` |
| `request_id` | `initial_grant_sub_xxx` |
| `created_at` | timestamp |

### 3) Intégration webhook

**Fichier** : `keybuzz-api/src/modules/billing/routes.ts`

Ajouté dans `handleSubscriptionChange()`, après la mise à jour de `tenants.plan` :

```
const { grantInitialKBActions } = await import('../../services/ai-actions.service');
const grantResult = await grantInitialKBActions(tenantId, plan, subscription.id);
```

**Event Stripe déclencheur** : `customer.subscription.created` / `customer.subscription.updated` (routé via `handleSubscriptionChange`)

### 4) Debug endpoint enrichi

**Fichier** : `keybuzz-client/app/api/debug-profile-billing/route.ts`

Ajout d'un appel à `/ai/wallet/status?tenantId=${tenantId}` pour inclure le solde réel du wallet dans la réponse debug.
