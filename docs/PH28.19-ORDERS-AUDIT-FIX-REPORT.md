## Objectifs

1. Page Commandes fonctionne pour tenant avec/sans commandes
2. Aucun redirect /login si user authentifié

4. Si API down : banner "API indisponible" + retry
5. Pas de données demo en session authentifiée

---

## Analyse

### Architecture Orders

```
Client page (/orders)
    ↓ fetch
Next.js API route (/api/orders)
    ↓ gets tenant from /tenant-context/me
    ↓ calls backend with X-Tenant-Id header
Backend (BACKEND_URL/api/v1/orders)
    ↓ returns orders filtered by tenant
```

### Bug Identifié

**Dans `app/orders/page.tsx`:**
```typescript
} catch (err) {
  console.error('[Orders] Fetch error:', err);
  setError((err as Error).message);
  // Fallback to mock in DEV only if API fails
  setOrders(MOCK_ORDERS);  // ← BUG: Affiche données mock !
}
```

Ce code fallback vers `MOCK_ORDERS` même pour un utilisateur authentifié, violant la règle "aucune donnée démo en session authentifiée".

---

## Corrections Appliquées

### 1. Suppression fallback MOCK_ORDERS

```typescript
// AVANT
setOrders(MOCK_ORDERS);

// APRÈS (PH28.19)
setOrders([]);  // Liste vide, pas de mock
```

### 2. Ajout loading state

```typescript
if (loading) {
  return (

  );
}
```

### 3. Ajout error banner

