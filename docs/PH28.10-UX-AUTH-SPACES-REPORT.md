## 1. Changements Effectués

### 1.1 Login - Email Inconnu (Partie A)

**Avant** : Message d'erreur technique si email non trouvé

**Après** : 
- Vérification préalable via `/api/auth/check-email`
- Si email inconnu → affichage écran "Aucun compte trouvé"
- Bouton "Créer un compte" → redirection `/signup?email=...`
- Email pré-rempli sur la page signup

**Code** : `app/login/page.tsx`
```typescript
// PH28.10: First check if email exists
const checkRes = await fetch(`${AUTH_BASE}/check-email`, { ... });
if (!checkData.exists) {
  setStep('not_found');
  return;
}
```

### 1.2 Message OTP Whitelist (Partie B)

**Avant** : "OTP non disponible pour cet email (mode test)"

**Après** : 
```
La connexion par code n'est pas encore disponible pour cet email.
Utilisez Google ou Microsoft, ou créez votre compte.
```

**Confirmation** : Aucune mention de "mode test" dans l'UI.

### 1.3 Spaces Empty State (Partie C)

**Avant** : "Impossible de charger vos espaces" (erreur rouge)

**Après** : 
- Si 0 espace : Empty state avec icône folder
- Titre : "Aucun espace pour le moment"
- Bouton principal : "Créer mon premier espace"
- Pas de redirection automatique vers onboarding

**Code** : `app/select-tenant/page.tsx`
```typescript
// PH28.10: Empty state - no red error, friendly message
if (showEmptyState) {
  return (
    <div className="p-8 text-center">
      <FolderOpen className="h-8 w-8 text-gray-400" />
      <h2>Aucun espace pour le moment</h2>
      <button onClick={handleCreateSpace}>Créer mon premier espace</button>
    </div>
  );
}
```

### 1.4 Signup - Email Support (Partie D)

- Support du paramètre `?email=` pour pré-remplir
- Message OTP whitelist humain (même que login)
- Email inconnu → continue normalement vers onboarding
