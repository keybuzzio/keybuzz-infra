## 4. FIX CAP 50 — DÉCISION UI

### Problème initial

```typescript
// AVANT (cap 50)
const stats = useMemo(() => ({
  total: conversations.length,  // ❌ Limité à 50 (pagination)
  open: conversations.filter(c => c.status === "open").length,
  pending: conversations.filter(c => c.status === "pending").length,
  unread: conversations.filter(c => c.unread).length,
}), [conversations]);
```

### Solution PH26.4B

```typescript
// APRÈS (source unique)
const stats = useMemo(() => {
  const localCounts = {
    total: conversations.length,
    open: conversations.filter(c => c.status === "open").length,
    pending: conversations.filter(c => c.status === "pending").length,
    unread: conversations.filter(c => c.unread).length,
  };
  
  // ⚠️ SOURCE OF TRUTH — API stats si disponible
  if (apiStats) {
    return {
      total: apiStats.conversations.total,      // ✅ Valeur réelle (pas cap 50)
      open: apiStats.conversations.open,        // ✅ Valeur réelle
      pending: apiStats.conversations.pending,  // ✅ Valeur réelle
      unread: localCounts.unread,               // ⚠️ Local (non fourni par API)
      displayedCount: localCounts.total,
      hasMore: apiStats.conversations.total > localCounts.total,
    };
  }
  
  return { ...localCounts, displayedCount: localCounts.total, hasMore: false };
}, [conversations, apiStats]);
```

### Comportement UI

| Cas | Affichage compteur | Affichage liste |
|-----|-------------------|-----------------|
| `total = 58` | **58** | 50 éléments (pagination) |
| `hasMore = true` | **58+** | Indicateur visuel |

L'utilisateur voit maintenant le **vrai total** (58), même si la liste est paginée à 50.
