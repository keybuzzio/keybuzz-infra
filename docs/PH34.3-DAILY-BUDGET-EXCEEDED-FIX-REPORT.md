# PH34.3 — Fix DAILY_BUDGET_EXCEEDED (DEV → PROD)

## Cause racine

Le `PLAN_BUDGETS_DAILY` dans `keybuzz-api/src/config/ai-budgets.ts` définissait les budgets quotidiens IA avec des clés **minuscules** (`'pro': 0.50`), alors que la valeur `plan` stockée dans `tenants.plan` est en **MAJUSCULES** (`'PRO'`).

Résultat : `PLAN_BUDGETS_DAILY['PRO']` → `undefined` → default `0` → **toutes les requêtes IA bloquées** pour les tenants PRO/AUTOPILOT.

## Fix appliqué

### Backend (`keybuzz-api`)

| Fichier | Modification |
|---------|-------------|
| `src/config/ai-budgets.ts` | Ajout clés MAJUSCULES (`'PRO': 5, 'AUTOPILOT': 20, 'ENTERPRISE': 100`) + augmentation budgets (Pro: $0.50→$5/day, Autopilot: $2→$20/day) |
| `dist/services/ai-credits.service.js` | `plan.toLowerCase()` pour lookup case-insensitive |
| `src/modules/ai/credits-routes.ts` | Endpoint debug `GET /ai/debug/budget?tenantId=...` |

### Client (`keybuzz-client`)

| Fichier | Modification |
|---------|-------------|
| `src/features/ai-ui/AISuggestionSlideOver.tsx` | Messages UX : "Budget IA quotidien atteint" / "L'IA n'est pas disponible sur Starter" |
