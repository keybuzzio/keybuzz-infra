# PH28.18 - Inbox API Fix + Filter Report

**Date:** 2026-02-03
**Status:** ✅ COMPLETED
**Environments:** DEV + PROD

---

## Objectifs

1. Supprimer l'état "API indisponible" quand l'API est réellement up
2. Assurer que le filtre "En attente" fonctionne correctement

---

## Analyse Root Cause

### Symptôme
L'Inbox affichait "API indisponible" et "Stats API indisponible" même pour des utilisateurs authentifiés avec un tenant.

### Cause Racine Identifiée
**Le tenant ID n'était pas stocké dans localStorage après signup/onboarding.**

Flow problématique:
1. User signup → tenant créé en base
2. Onboarding détecte le tenant → appelle `/api/auth/select-tenant`
3. Cookie `currentTenantId` défini côté serveur ✅
4. **Mais `lastTenantId` dans localStorage n'était JAMAIS défini** ❌
5. Inbox charge → appelle `getCurrentTenantId()` → retourne `null`
6. Sans tenant ID → pas d'appel API → fallback "demo" (maintenant "API indisponible")

### Preuve: API Logs
Aucune requête vers `/stats/conversations` ou `/messages/conversations` dans les logs API lors du chargement Inbox - confirmant que le client n'appelait pas l'API.

---

## Fix Appliqué

### Fichier: `app/onboarding/page.tsx`

```typescript
// AVANT
console.log('[Onboarding] Tenant already exists, redirecting:', tenant.name);
await fetch('/api/auth/select-tenant', {...});
router.replace('/');

// APRÈS (PH28.18)
import { setLastTenant } from '@/src/lib/session';
// ...
console.log('[Onboarding] Tenant already exists, redirecting:', tenant.name);
setLastTenant(tenant.id, tenant.name, tenant.id);  // ← AJOUTÉ
await fetch('/api/auth/select-tenant', {...});
router.replace('/');
```

Le fix ajoute `setLastTenant()` pour stocker le tenant dans localStorage AVANT la redirection.

## Filtre "En attente"

### Vérification
Le filtre "En attente" fonctionne déjà correctement:
- Default: filtre `pending` sélectionné
- Logic: `matchesStatus = statusFilter === "all" || c.status === statusFilter`
- Stats API retourne `pending: 0` pour le tenant test → 0 conversations "En attente" est correct

### Comportement Attendu
| Situation | Affichage |
|-----------|-----------|
| pending > 0 | Liste des conversations en attente |
| pending = 0 + dataSource = api | Empty state "✅ Excellent travail !" |
| dataSource = error | Liste vide + "API indisponible" |
