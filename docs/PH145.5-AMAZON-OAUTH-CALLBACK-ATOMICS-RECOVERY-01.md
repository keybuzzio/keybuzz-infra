```typescript

  tenantId: oauthState.tenantId,
```

## 3. Causes racine exactes

| # | Cause | Impact |
|---|-------|--------|
| **C1** | `WHERE tenant_id = $1 AND provider = 'amazon'` sans filtre pays | Active en masse TOUS les channels |
| **C2** | UPDATE ne set pas `inbound_email` | Email invisible sur la carte |
| **C3** | UPDATE ne set pas `connection_ref` | Pas de lien channel ↔ connection |
| **C4** | `countries: ["FR"]` hardcode | Seul FR recoit une adresse inbound |
| **C5** | Status `DRAFT` jamais mis a `READY` | Status endpoint retourne `connected: false` |

## 4. Correction callback (PH145.5)

### Nouveau code (amazon.routes.ts)

Le bloc post-OAuth (anciennement lignes 330-373) est remplace par :

```typescript
// PH145.5: Atomic channel activation with multi-country inbound provisioning
try {
  // Step 1: Get all non-removed Amazon channels for this tenant
  const channelsResult = await productDb.query(
    `SELECT country_code FROM tenant_channels
     WHERE tenant_id = $1 AND provider = 'amazon' AND status != 'removed'`,
    [oauthState.tenantId]
  );
  const countries: string[] = channelsResult.rows.map((r: any) => r.country_code);
  if (countries.length === 0) countries.push("FR");

  // Step 2: Create inbound connection + addresses for ALL countries
  const inboundConn = await ensureInboundConnection({
    tenantId: oauthState.tenantId,
    marketplace: "amazon",
    countries,
  });

  // Step 3: Set connection status to READY (OAuth completed)
  await prisma.inboundConnection.update({
    where: { id: inboundConn.id },
    data: { status: 'READY' },
  });

  // Step 4: Get all created/existing addresses
  const addresses = await prisma.inboundAddress.findMany({
    where: { connectionId: inboundConn.id },
  });

  // Step 5: Atomically update EACH channel with its specific address
  let activatedCount = 0;
  for (const addr of addresses) {
    const res = await productDb.query(
      `UPDATE tenant_channels
       SET status = 'active', inbound_email = $1, connection_ref = $2,
           activated_at = NOW(), connected_at = NOW(), updated_at = NOW()
       WHERE tenant_id = $3 AND provider = 'amazon' AND country_code = $4
         AND status != 'removed'`,
      [addr.emailAddress, inboundConn.id, oauthState.tenantId, addr.country]
    );
    activatedCount += res.rowCount || 0;
  }

  // Step 6: Fallback - create FR channel if none existed
  if (activatedCount === 0 && addresses.length > 0) {
    const frAddr = addresses.find((a: any) => a.country === 'FR') || addresses[0];
    await productDb.query(
      `INSERT INTO tenant_channels (...)
       VALUES ($1, 'amazon', $2, $3, $4, 'active', $5, $6, NOW(), NOW())
       ON CONFLICT ... DO UPDATE SET ...`,
      [oauthState.tenantId, frAddr.country, ...]
    );
    activatedCount = 1;
  }
} catch (channelErr) {
  console.warn("[Amazon OAuth] PH145.5: Failed:", channelErr);
}
```

### Changements cles

| Avant | Apres |
|-------|-------|
| `countries: ["FR"]` | Lit dynamiquement les country_code existants |
| Status reste `DRAFT` | Force `READY` apres OAuth |
| `UPDATE ... WHERE provider = 'amazon'` | `UPDATE ... WHERE country_code = $4` |
| Pas de `inbound_email` | SET `inbound_email = $1` par pays |
| Pas de `connection_ref` | SET `connection_ref = $2` par pays |
| 1 UPDATE pour tous | 1 UPDATE par pays (atomique) |

## 5. Correction multi-pays

Le fix corrige naturellement le multi-pays :

1. **Lecture des channels** : `SELECT country_code FROM tenant_channels WHERE tenant_id = $1 AND provider = 'amazon' AND status != 'removed'`

Pays supportes : FR, DE, ES, IT, BE, NL, PL, IE, et tout autre pays dans `tenant_channels`.

## 9. Flux OAuth complet post-fix

```
1. User ajoute Amazon depuis le catalogue
   → addChannel() met status = 'pending' pour chaque pays choisi
   → Card montre "En attente" + bouton "Connecter Amazon"

2. User clique "Connecter Amazon"
   → BFF /api/amazon/oauth/start
   → Backend genere Amazon auth URL (LWA) avec state lie au tenant

3. User autorise sur Amazon Seller Central
   → Amazon redirige vers platform-api callback

4. Backend callback (PH145.5):
   a. completeAmazonOAuth() — tokens Vault + MarketplaceConnection CONNECTED
   b. Lit les channels existants → countries = [FR, DE, BE, IE, ...]
   c. ensureInboundConnection({ countries }) → cree TOUTES les adresses
   d. prisma.inboundConnection.update({ status: 'READY' })
   e. Pour CHAQUE adresse:
      UPDATE tenant_channels SET status='active', inbound_email=addr, connection_ref=connId
      WHERE tenant_id=$1 AND country_code=$2
   f. Seuls les channels ayant une adresse sont actives

5. Status endpoint retourne connected: true (inbound_connections READY)

6. Channels page affiche:
   - Channels avec connection_ref → "Connecte" + email visible
   - Channels sans connection_ref → "OAuth requis" + bouton

7. Orders page affiche marketplace + bouton orange
```
