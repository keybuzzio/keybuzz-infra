**Mitigation** : promouvoir le backend PROD vers DEV si necessaire.

## 8. PRE-CONDITIONS POUR LA RECONSTRUCTION

### Commit discipline (E6)

Avant chaque micro-phase :

1. `git status` propre (sauf fichiers du scope)
2. `git diff --stat` coherent avec le micro-scope
3. `pre-build-check.sh` doit passer sur le bastion
4. Build Docker depuis Git uniquement
5. Deploy + validation reelle

### Synchronisation bastion

Le bastion doit recevoir les commits AVANT le build :

1. Push local vers remote (`origin main`)
2. Sur le bastion : `git fetch origin` puis `git merge` ou `git checkout` le bon commit
3. `pre-build-check.sh` valide
4. `docker build --no-cache`
