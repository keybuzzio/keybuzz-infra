## 4. Cause racine unique

**Les changements Shopify (PH-SHOPIFY-02 et PH-SHOPIFY-02.1) ont été appliqués sur le bastion via scripts Python mais jamais synchronisés vers le workspace local.**

Le fichier `channels/page.tsx` sur le bastion avait été écrasé par des builds ultérieurs (PH145.x) qui partaient du workspace local, lequel ne contenait aucune référence à Shopify.

Résultat : bien que le logo et les BFF routes existaient sur le bastion, le fichier `channels/page.tsx` déployé ne les utilisait pas.

## 5. Fix appliqué

### Fichiers créés


| Fichier                               | Rôle                                            |
| ------------------------------------- | ----------------------------------------------- |
| `public/marketplaces/shopify.svg`     | Logo Shopify (vert #95BF47 + violet #5C6AC4)    |
| `src/services/shopify.service.ts`     | Service client (getStatus, connect, disconnect) |
| `app/api/shopify/status/route.ts`     | BFF route GET proxy vers API backend            |
| `app/api/shopify/connect/route.ts`    | BFF route POST proxy vers API backend           |
| `app/api/shopify/disconnect/route.ts` | BFF route POST proxy vers API backend           |


### Fichier modifié : `app/channels/page.tsx`


| Modification       | Description                                                                              |
| ------------------ | ---------------------------------------------------------------------------------------- |
| Import             | Ajout import `shopify.service`                                                           |
| PROVIDER_LOGOS     | Ajout `shopify: "/marketplaces/shopify.svg"`                                             |
| State variables    | `shopifyStatus`, `showShopifyModal`, `shopifyDomain`, `shopifyConnecting`                |
| Status refresh     | `getShopifyStatus()` dans `refreshProviderStatus`                                        |
| Callback detection | Détection `?shopify_connected=true` / `?shopify_error=`*                                 |
| Handlers           | `handleShopifyConnect()`, `handleShopifyDisconnect()`                                    |
| Catalog click      | Interception `provider === 'shopify'` → ouvre modal au lieu de `addChannel`              |
| Channel cards      | Boutons "Connecter Shopify" (pending) + badge "Connecté" (active)                        |
| Remove handler     | Auto-disconnect Shopify au retrait du canal                                              |
| Modal              | Modal "Connexion Shopify" avec input domaine `.myshopify.com`, boutons Annuler/Connecter |
