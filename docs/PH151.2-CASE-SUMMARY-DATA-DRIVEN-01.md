**Date** : 13 avril 2026

**Rollback** : `v3.5.62-ph151.1-inbox-clean-dev`

---

## Objectif

Remplacer le résumé approximatif (texte extrait du premier message) par un résumé fiable basé sur des données réelles de la conversation.

---

## Ancien vs Nouveau résumé

### AVANT (PH151.1)

| Section | Logique | Problème |
|---|---|---|
| Problème initial | `summarizeContent(firstMsg, 120)` — 1ère ligne "meaningful" | Texte brut tronqué, pas de classification |
| Actions | Regex sur messages Amazon auto | "Remboursement détecté" sans contexte, pas de notes internes |
| État actuel | Map `status → label` | Ignore SAV, escalation, assignation |

### APRÈS (PH151.2)

| Section | Logique | Source de données |
|---|---|---|
| **Catégorie** (icône + label) | `detectProblemCategory()` — analyse subject + fallback content | `conversation.subject`, regex multi-langue |
| **Demande** | Extrait du premier message client, en italique | `messages[0].content` (après filtre greetings) |
| **Actions** | Comptage réel par type d'acteur | `messages.direction`, `authorType`, `messageSource` |
| **État** | Multi-sources : status + SAV + escalation + assignation | `status`, `savStatus`, `escalationStatus`, `assignedType` |
| **Timing** | Âge calculé depuis premier message | `messages[0].timestamp` |

## Actions détectées

| Action | Source | Badge |
|---|---|---|
| `X réponses agent` | `direction=outbound, authorType!=ai, visibility!=internal_note` | bleu |
| `X réponses IA` | `authorType=ai` ou `messageSource=AI` | bleu |
| Remboursement initié | regex sur messages Amazon auto | ambre |
| Retour autorisé | regex sur messages Amazon auto | ambre |
| Réclamation ouverte | regex sur messages Amazon auto | ambre |
| Fournisseur contacté | `messageSource=SUPPLIER_*` | vert |
| `X notes internes` | `visibility=internal_note` | gris |

## Limites restantes

1. **Pas de fallback LLM** : si aucune catégorie n'est détectée via subject/content, affiche "Demande client" générique. Un appel LLM pourrait affiner, mais le prompt interdit LLM pour le résumé principal.
2. **Dépend du subject** : si le subject est vide ou générique (ex: "Message de Client Amazon"), la classification retombe sur l'analyse du premier message.
3. **Pas de statut commande** : le résumé ne montre pas le statut de livraison de la commande (cette info est déjà visible dans la section Commande en-dessous).
