## 1. PROD - Debug /signup

### Problème identifié
La page `/signup` redirigeait automatiquement vers `/login` car `AuthGuard` ne reconnaissait pas `/signup` comme route publique.

**Fichier**: `src/components/auth/AuthGuard.tsx`
```typescript
// AVANT
const PUBLIC_ROUTES = ["/login", "/auth/callback", "/pricing"];

// APRÈS
const PUBLIC_ROUTES = ["/login", "/auth/callback", "/pricing", "/signup"];
```

### Fix appliqué
- Ajout de `/signup` à la liste `PUBLIC_ROUTES` dans `AuthGuard.tsx`
- Rebuild et déploiement de l'image PROD

## 2. DEV - Debug "Impossible de charger vos espaces"

### Problème identifié
L'API DEV retournait des erreurs 500 sur `/tenant-context/me` et `/auth/tenants`:

```
[Auth Tenants] API error: 500 { error: 'Failed to get user' }
password authentication failed for user "keybuzz_api_dev"
```

**Cause racine**: Le mot de passe de l'utilisateur PostgreSQL `keybuzz_api_dev` avait été modifié directement sur le serveur DB, causant une désynchronisation avec le secret Vault/K8s.

### Fix appliqué
Reset du mot de passe PostgreSQL sur le leader Patroni (`db-postgres-03`):

```sql
ALTER USER keybuzz_api_dev WITH PASSWORD 'IfVXI5kTGZ87Rh4LH9kVY3sGgDq3jNwR';
```

Puis rollout du pod API DEV:
```bash
kubectl rollout restart deployment/keybuzz-api -n keybuzz-api-dev
```
