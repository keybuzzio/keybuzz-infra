## 5. Cas Tracking — Analyse detaillee

### Pourquoi carrier/tracking_code sont toujours null ?

1. L'API SP-API **Orders** (`GET /orders/v0/orders/{orderId}`) retourne les commandes mais **NE RETOURNE PAS** le numero de suivi.

2. Le numero de suivi est disponible via :
   - **Shipping API** (`POST /shipping/v2/shipments`) — pour les expeditons via Amazon Buy Shipping
   - **Reports API** — rapport `GET_FLAT_FILE_ALL_ORDERS_DATA_BY_ORDER_DATE_GENERAL` contient le tracking
   - **Fulfillment Outbound Shipments** — pour FBA uniquement

3. Le worker d'import (`amazon-orders-worker`) appelle l'Orders API et mappe les champs, mais le tracking n'est PAS dans la reponse.

4. Le CronJob `amazon-reports-tracking-sync` (toutes les 6h) est cense synchroniser les tracking via Reports, mais les colonnes `carrier` et `tracking_code` restent a 0/11 344 — le worker **n'ecrit probablement pas dans ces colonnes**.

### Impact IA

- Quand un client demande "ou est mon colis ?", l'IA **ne peut pas** donner le numero de suivi
- L'IA propose "proposer de verifier dans Seller Central" (fallback code)
- La fenetre de livraison estimee (`EarliestDeliveryDate/LatestDeliveryDate`) est dans `raw_data` mais **non injectee**

## 9. Synthese

### Ce qui fonctionne bien
- orderRef, status, totalAmount, currency, productTitle — 100% present et injecte
- deliveryStatus, fulfillmentChannel, purchaseDate — 100% present et injecte
- Contexte fournisseur (supplier_cases) fonctionne en parallele

### Ecarts critiques
1. **Tracking = 0/11 344** — les colonnes existent mais ne sont jamais remplies. L'API SP-API Orders ne retourne pas le tracking. Le CronJob Reports ne les ecrit pas.
2. **Fenetre de livraison disponible mais non injectee** — `EarliestDeliveryDate` / `LatestDeliveryDate` sont dans raw_data, pas dans le prompt IA.
3. **MarketplaceId non injecte** — disponible dans raw_data, pas dans OrderContext.
4. **Octopia sans contexte commande** — aucune conversation Octopia n'a de commande liee.
5. **Retours Amazon non croises** — table `amazon_returns` existe mais n'est pas consultee par l'IA.
6. **buyerName toujours "Client Amazon"** — anonymise a l'import, PII Amazon non dispo depuis 2023.

### Conclusion

Le prompt IA recoit **10 des 18 champs demandes** (55%). Les champs les plus impactants manquants (`deliveryDate`, `marketplace`, `tracking`) sont disponibles dans `raw_data` et peuvent etre mappes avec un patch minimal (R1-R4). Le tracking reel necessiterait un travail sur le worker Reports.
