## 3. Cause racine exacte

### Bug 1 : "0 inclus dans votre plan" (Page Canaux)

- **Cause** : La page Canaux utilisait `billing.plan_included` depuis `/api/channels/billing` qui retourne 0.
- **Source correcte** : `billingCompute.channelsIncluded` depuis `/api/channels/billing-compute` retourne 5.
- **Fix** : Utiliser `billingCompute.channelsIncluded` quand disponible, sinon fallback `billing.plan_included`.

### Bug 2 : "0 canaux connectés" (Page Options)

- **Cause** : La page Options comptait les canaux connectés en fetchant manuellement `/api/amazon/status` et `/api/octopia/status`. Ces endpoints retournent un statut global (1 connexion Amazon globale, pas par pays), donc le compteur ne reflétait pas les 4 canaux individuels.
- **Source correcte** : `/api/channels/list` retourne la liste réelle de tous les canaux connectés.
- **Fix** : Remplacer le comptage manuel Amazon/Octopia par un appel à `/api/channels/list`.

### Bug 3 : "canalx" et "canalaux" (pluralisation)

- **Cause** : Concaténation naïve de `canal` + suffixe.
  - `canal{n > 1 ? 'x' : ''}` → "canalx" (Options)
  - `canal{n > 1 ? "aux" : ""}` → "canalaux" (Canaux)
- **Fix** : Remplacement par ternaire complet : `n > 1 ? 'canaux' : 'canal'`.
