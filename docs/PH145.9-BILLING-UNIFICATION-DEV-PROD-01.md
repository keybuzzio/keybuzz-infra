# PH145.9 — Unification Module Billing KBActions DEV/PROD

> Date : 12 avril 2026
> Tag : `v3.5.254-ph145.9-billing-unified-dev`
> Image précédente : `v3.5.253-ph145.8-resolve-dev`
> Scope : **DEV uniquement** (aucun push PROD)
> Statut : **DEPLOYED + VALIDATED**

---

## 1. OBJECTIF

Unifier le module de facturation KBActions entre DEV et PROD :

- Supprimer les "Outils de Test (DEV ONLY)" visible uniquement en DEV
- Activer le module d'achat Stripe **partout** (DEV + PROD)
- Uniformiser le wording "KBActions" (au lieu de "actions IA" ou "credits")

---

## 2. ÉTAT AVANT (captures navigateur)

### DEV (`client-dev.keybuzz.io/billing/ai/manage`)


| Élément       | État                                                                           |
| ------------- | ------------------------------------------------------------------------------ |
| Header        | "Configuration et outils **(DEV)** [owner@ecomlg-001]"                         |
| Bloc jaune    | **"Outils de Test (DEV ONLY)"** avec 3 boutons fake (+100 KBA, -10 KBA, Vider) |
| Module Stripe | **ABSENT** — masqué par condition `{!isDev && ...}`                            |
| Wording       | Mélange "actions IA" / "credits" / "KBActions"                                 |


### PROD (`client.keybuzz.io/billing/ai/manage`)


| Élément       | État                                                                          |
| ------------- | ----------------------------------------------------------------------------- |
| Header        | "Configuration et outils [owner@ecomlg-001]"                                  |
| Bloc jaune    | Absent (pas de DEV tools en PROD)                                             |
| Module Stripe | **"Acheter des credits"** — bouton **"Bientôt disponible (Stripe)"** DISABLED |
| Wording       | "credits" au lieu de "KBActions"                                              |


### Problème racine

Le fichier `app/billing/ai/manage/page.tsx` utilisait un flag `isDev` (hostname detection) pour :

- Afficher les outils DEV en DEV
- Masquer le module Stripe en DEV
- Le module Stripe en PROD avait un bouton disabled "Bientôt disponible"

## 7. CORRECTIONS BONUS (pendant le build)

Le build initial a échoué car le bastion avait une version plus récente d'`InboxTripane.tsx`
qui passait des props non déclarées dans l'interface du SlideOver :


| Prop             | Type                              | Raison                         |
| ---------------- | --------------------------------- | ------------------------------ |
| `onDirectSend`   | `(text: string) => Promise<void>` | Autopilot direct send          |
| `initialDraft`   | `AutopilotDraft | null`           | Draft autopilot initial        |
| `autoOpen`       | `boolean`                         | Auto-ouverture panel           |
| `AutopilotDraft` | interface exportée                | Type pour les drafts autopilot |


Ces ajouts sont compatibles backward — toutes les props sont optionnelles.
