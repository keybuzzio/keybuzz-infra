## 7. RISQUES RESIDUELS

### Risque 1 : Fichiers C avec modifications mineures non detectees

51 fichiers C restent a leur version release. Il est theoriquement possible qu'un fichier C contienne une modification mineure (non detectable par analyse de symboles) qui etait presente dans v3.5.63. Cependant :

- L'ajout de `types.ts` et `constants.ts` (seules dependances de type) devrait couvrir les cas de compilation
- Les fichiers C sont principalement des changements cosmetiques (textes FR, espacement, etc.)

**Impact : FAIBLE** ‚Äî n'affecte que le rendu visuel, pas la compilation.

### Risque 2 : Chunk register/page.tsx

La preuve compiled montre `siret`, `companyPhone`, `zipCode` dans le chunk `register/page-b1fad2934a45f798.js`. Le fichier `app/register/page.tsx` n'est pas dans les 17 fichiers A. Deux possibilites :

- Next.js a bundle du code settings dans le chunk register (code splitting)
- Le register/page.tsx utilise aussi BusinessProfile

L'ajout des champs a BusinessProfile (D1-1) est **additif** et ne casse pas le code existant de register. La version release de register compilera correctement meme avec le nouveau type.

**Impact : NUL** ‚Äî ajouter des champs a une interface TypeScript est retrocompatible.

### Risque 3 : Difference accents dans constants.ts

Le diff de `constants.ts` (D1-2) inclut un nettoyage d'accents (`Fete du Travail` au lieu de `F√™te du Travail`, `Noel` au lieu de `No√´l`). Ce sont des donnees affichees uniquement les jours feries.

**Impact : NEGLIGEABLE** ‚Äî difference cosmetique sans impact fonctionnel.

### Risque 4 : useCurrentPlan.tsx reordonnancement

Le diff entre release et ph152.6 pour `useCurrentPlan.tsx` est un simple reordonnancement du champ `hasAgentKeybuzzAddon` dans l'interface et les useState. Aucun changement fonctionnel. La version release est suffisante.

**Impact : NUL** ‚Äî le reordonnancement n'affecte pas les types exportes.
