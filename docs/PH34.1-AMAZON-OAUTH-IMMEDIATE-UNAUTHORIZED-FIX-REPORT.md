# PH34.1 — Amazon "OAuth Unauthorized" immédiat — RCA + Fix

**Date** : 2026-02-17
**Statut** : Déployé DEV + PROD [PROD-APPROVED]

---

## Symptôme

Clic sur "Connecter Amazon" → message d'erreur immédiat "Erreur OAuth: Unauthorized" sans jamais atteindre la page Amazon Seller Central.

---

## Endpoint exact

```
Bouton "Connecter Amazon" (app/channels/page.tsx)
  → startAmazonOAuth() (src/services/amazon.service.ts)
    → POST /api/amazon/oauth/start (BFF Next.js)
      → POST ${BACKEND_URL}/api/v1/marketplaces/amazon/oauth/start
```

---

## Cause racine

| Composant | URL | Route Amazon | Résultat |
|-----------|-----|-------------|----------|
| keybuzz-api (port 3001) | `BACKEND_URL` | **N'EXISTE PAS** | **HTTP 402** (paywall guard) |
| keybuzz-backend (port 4000) | `AMAZON_BACKEND_URL` | **EXISTE** | **HTTP 200** + authUrl |

Le BFF utilisait `BACKEND_URL` = `http://keybuzz-api....:3001` pour appeler les routes Amazon. Mais `keybuzz-api` n'a PAS ces routes (aucun module compat/proxy). La requête était interceptée par le paywall guard global (`onRequest` hook) qui retournait **HTTP 402 Payment Required**.

Les routes Amazon n'existent que dans `keybuzz-backend` (port 4000), le legacy backend qui gère les intégrations marketplace.

**Ce n'est PAS un problème de secrets, de paywall, ou d'auth** — c'est un problème de routing réseau.

---

## Fix

### Code modifié

5 fichiers BFF Amazon :

| Fichier | Modification |
|---------|-------------|
| `app/api/amazon/oauth/start/route.ts` | `BACKEND_URL` → `AMAZON_BACKEND_URL \|\| BACKEND_URL` |
| `app/api/amazon/status/route.ts` | idem |
| `app/api/amazon/disconnect/route.ts` | idem |
| `app/api/amazon/inbound-address/route.ts` | idem |
| `app/api/amazon/inbound-address/send-validation/route.ts` | idem |

### Env var ajoutée

| Env | DEV | PROD |
|-----|-----|------|
| `AMAZON_BACKEND_URL` | `http://keybuzz-backend.keybuzz-backend-dev.svc.cluster.local:4000` | `http://keybuzz-backend.keybuzz-backend-prod.svc.cluster.local:4000` |
| `BACKEND_URL` (inchangé) | `http://keybuzz-api...:3001` | `http://keybuzz-api...:80` |
