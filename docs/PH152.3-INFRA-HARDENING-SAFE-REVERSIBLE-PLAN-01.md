## 1. PLAN iptables SAFE (dead man's switch)

### 1.1 Principe

Chaque application de regles iptables utilise un **dead man's switch** : les regles anciennes sont automatiquement restaurees apres 120 secondes sauf si l'operateur confirme explicitement. Si SSH est perdu, le rollback est automatique.

### 1.2 Script safe-iptables.sh

Ce script doit etre SCP vers chaque serveur cible AVANT execution.

```bash
#!/bin/bash
# safe-iptables.sh — Dead man's switch iptables
# Usage: ./safe-iptables.sh [apply|confirm|abort]
# Sequence: apply → (test SSH) → confirm | abort

set -euo pipefail

BACKUP="/root/iptables-backup-$(date +%Y%m%d-%H%M%S).rules"
TIMEOUT=120  # secondes avant rollback automatique
SELF_IP=$(hostname -I | awk '{print $1}')

case "${1:-}" in

apply)
    # 1. Backup
    iptables-save > "$BACKUP"
    echo "[BACKUP] Sauvegarde: $BACKUP"

    # 2. Programmer rollback automatique dans 120s
    echo "iptables-restore < $BACKUP && echo '[ROLLBACK] Regles restaurees automatiquement'" | at now + 2 minutes 2>/dev/null
    AT_JOB=$(atq | tail -1 | awk '{print $1}')
    echo "[DEADMAN] Rollback programme dans ${TIMEOUT}s (at job $AT_JOB)"
    echo "$AT_JOB" > /tmp/iptables-deadman-job

    # 3. Appliquer les nouvelles regles
    iptables -F INPUT

    # Loopback
    iptables -A INPUT -i lo -j ACCEPT

    # Connexions etablies (CRITIQUE pour SSH en cours)
    iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

    # SSH depuis reseau prive + bastion
    iptables -A INPUT -s 10.0.0.0/16 -p tcp --dport 22 -j ACCEPT
    iptables -A INPUT -s 46.62.171.61/32 -p tcp --dport 22 -j ACCEPT
    # SSH depuis IP publiques des masters (au cas ou)
    iptables -A INPUT -s 91.98.124.228/32 -p tcp --dport 22 -j ACCEPT

    # Reseau prive Hetzner
    iptables -A INPUT -s 10.0.0.0/16 -j ACCEPT

    # K8s pods (Calico)
    iptables -A INPUT -s 10.244.0.0/16 -j ACCEPT

    # K8s services (ClusterIP)
    iptables -A INPUT -s 10.96.0.0/12 -j ACCEPT

    # ICMP
    iptables -A INPUT -p icmp -j ACCEPT

    # Drop
    iptables -A INPUT -j DROP

    echo "[APPLY] Regles appliquees. TESTEZ SSH maintenant."
    echo "[APPLY] Vous avez ${TIMEOUT}s pour executer: ./safe-iptables.sh confirm"
    echo "[APPLY] Si vous ne confirmez pas, rollback automatique."
    ;;

confirm)
    # Annuler le dead man's switch
    if [ -f /tmp/iptables-deadman-job ]; then
        AT_JOB=$(cat /tmp/iptables-deadman-job)
        atrm "$AT_JOB" 2>/dev/null && echo "[CONFIRM] Dead man's switch annule (job $AT_JOB)"
        rm -f /tmp/iptables-deadman-job
    fi
    echo "[CONFIRM] Regles confirmees. Elles restent actives."
    echo "[CONFIRM] Pour persister au reboot: iptables-save > /etc/iptables/rules.v4"
    ;;

abort)
    # Restaurer immediatement
    LATEST_BACKUP=$(ls -t /root/iptables-backup-*.rules 2>/dev/null | head -1)
    if [ -n "$LATEST_BACKUP" ]; then
        iptables-restore < "$LATEST_BACKUP"
        echo "[ABORT] Regles restaurees depuis $LATEST_BACKUP"
    fi
    if [ -f /tmp/iptables-deadman-job ]; then
        AT_JOB=$(cat /tmp/iptables-deadman-job)
        atrm "$AT_JOB" 2>/dev/null
        rm -f /tmp/iptables-deadman-job
    fi
    ;;

*)
    echo "Usage: $0 [apply|confirm|abort]"
    echo ""
    echo "  apply   - Backup + appliquer regles + programmer rollback auto 120s"
    echo "  confirm - Annuler le rollback auto (garder les nouvelles regles)"
    echo "  abort   - Restaurer immediatement les regles precedentes"
    ;;
esac
```

### 1.3 Garanties de securite


| Protection                     | Mecanisme                                                                                      |
| ------------------------------ | ---------------------------------------------------------------------------------------------- |
| Perte SSH impossible           | `ESTABLISHED,RELATED` en premier = la session SSH ouverte n'est jamais coupee                  |
| Rollback automatique si bloque | `at now + 2 minutes` restore le backup                                                         |
| Rollback manuel immediat       | `./safe-iptables.sh abort`                                                                     |
| Pas de persistence             | Les regles ne survivent PAS au reboot tant que `confirm` + `iptables-save` n'ont pas ete faits |
| Backup horodate                | Chaque `apply` cree un backup unique `/root/iptables-backup-YYYYMMDD-HHMMSS.rules`             |

## 2. SCENARIO SAFE — Procedure complete pas a pas

### 2.1 Prerequis (avant de toucher le premier serveur)

```
1. Ouvrir 2 sessions SSH vers le bastion (46.62.171.61)
2. Depuis la session 1 : ssh vers le serveur cible (ex: 10.0.0.134)
3. Depuis la session 2 : garder SSH vers le bastion (parachute)
4. SCP le script : scp safe-iptables.sh 10.0.0.134:/root/
5. chmod +x /root/safe-iptables.sh
```

### 2.2 Application (session 1, sur le serveur cible)

```
# Terminal 1 (sur le serveur cible)
./safe-iptables.sh apply
# → Affiche: "Regles appliquees. Vous avez 120s pour confirmer."
```

### 2.3 Test (session 2, depuis le bastion)

```
# Terminal 2 (depuis le bastion, NOUVELLE connexion)
ssh 10.0.0.134 'echo SSH_OK'
# → Si "SSH_OK" : acces confirme

# Tester le service local
curl -s http://10.0.0.134:9000/minio/health/live  # MinIO
curl -s http://10.0.0.150:8200/v1/sys/health       # Vault
# etc.

# Tester depuis un pod K8s
kubectl run test --rm -it --restart=Never --image=curlimages/curl -- \
  curl -s http://10.0.0.134:9000/minio/health/live
```

### 2.4 Confirmation (session 1, si tout OK)

```
# Terminal 1 (sur le serveur cible)
./safe-iptables.sh confirm
# → Annule le dead man's switch
```

### 2.5 Persistence (seulement apres validation complete)

```
# Persister au reboot (UNIQUEMENT apres confirmation)
iptables-save > /etc/iptables/rules.v4
```

### 2.6 Echec — 3 niveaux de rollback


| Situation                                | Action                                                        | Delai     |
| ---------------------------------------- | ------------------------------------------------------------- | --------- |
| SSH fonctionne mais service KO           | `./safe-iptables.sh abort` (terminal 1)                       | Immediat  |
| SSH ne fonctionne plus                   | Attendre 120s → rollback automatique (dead man's switch)      | 2 min max |
| Serveur bloque (ni SSH ni rollback auto) | Reboot depuis Hetzner Cloud Console (iptables non persistees) | 3-5 min   |


### 2.7 Simulation de blocage (test dry-run)

Avant le premier vrai serveur, on peut tester le mecanisme sur le serveur le moins critique (MinIO) :

```
1. SSH vers 10.0.0.134
2. ./safe-iptables.sh apply
3. NE PAS confirmer
4. Attendre 120s
5. Verifier que les regles sont automatiquement restaurees : iptables -L INPUT -n
6. Resultat attendu : policy ACCEPT, aucune regle
```

### 4.1 Etat actuel verifie


| Service            | URL                         | Reponse                         | Auth applicative                            |
| ------------------ | --------------------------- | ------------------------------- | ------------------------------------------- |
| Studio Client DEV  | `studio-dev.keybuzz.io`     | 307 → `/login`                  | OUI (redirect login)                        |

| Studio API PROD    | `studio-api.keybuzz.io`     | 404                             | OUI (meme code)                             |


Les services ont deja une protection applicative (auth session). Pas d'acces libre aux donnees.

### 4.2 Recommandation : IP whitelist via Ingress annotation

Au lieu de supprimer les Ingress, les restreindre aux IPs autorisees :

```yaml
metadata:
  annotations:
    nginx.ingress.kubernetes.io/whitelist-source-range: "46.62.171.61/32,10.0.0.0/16"
```

Cette annotation limite l'acces aux seules IPs :

- `46.62.171.61` (bastion — IP publique)
- `10.0.0.0/16` (reseau prive Hetzner)

### 4.3 Commandes (a executer quand autorise)

```bash
# Studio Client DEV
kubectl annotate ingress keybuzz-studio -n keybuzz-studio-dev \
  nginx.ingress.kubernetes.io/whitelist-source-range="46.62.171.61/32,10.0.0.0/16" --overwrite

# Studio Client PROD
kubectl annotate ingress keybuzz-studio -n keybuzz-studio-prod \
  nginx.ingress.kubernetes.io/whitelist-source-range="46.62.171.61/32,10.0.0.0/16" --overwrite

# Studio API DEV
kubectl annotate ingress keybuzz-studio-api -n keybuzz-studio-api-dev \
  nginx.ingress.kubernetes.io/whitelist-source-range="46.62.171.61/32,10.0.0.0/16" --overwrite

# Studio API PROD
kubectl annotate ingress keybuzz-studio-api -n keybuzz-studio-api-prod \
  nginx.ingress.kubernetes.io/whitelist-source-range="46.62.171.61/32,10.0.0.0/16" --overwrite
```

### 4.4 Rollback

```bash
# Retirer la restriction
kubectl annotate ingress keybuzz-studio -n keybuzz-studio-dev \
  nginx.ingress.kubernetes.io/whitelist-source-range- --overwrite
# Repeter pour chaque ingress
```

### 4.5 Validation


| 3     | Verifier la syntaxe : `vault server -config=/etc/vault.d/vault.hcl -dev` (dry-run)               | Syntaxe OK               |
| 4     | Restart Vault : `systemctl restart vault`                                                        | —                        |
