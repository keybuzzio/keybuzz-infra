# PH33.16C — Trial end = current_period_end (DEV + PROD)

## Objectif
Aligner `trialEndsAt` sur la source de vérité Stripe (`billing_subscriptions.current_period_end`) au lieu du fallback `tenants.created_at + 14j`.

## Changement
**Fichier modifié** : `keybuzz-api/src/services/entitlement.service.ts`

### Logique `trialEndsAt` (PH33.16C)

| Cas | Source | Quand |
|-----|--------|-------|
| **A** : tenant a une `billing_subscriptions` row | `billing_subscriptions.current_period_end` | Subscription Stripe existe |
| **B** : tenant SANS `billing_subscriptions` | `tenants.created_at + 14 jours` | Aucune subscription en DB |

### Changement clé vs PH33.16B
- PH33.16B : chaîne `trial_end > current_period_end > created_at+14d` (avec détection dynamique colonne `trial_end`)
- PH33.16C : simplifié à `current_period_end` si subscription existe, sinon `created_at+14d`
- Suppression de la détection dynamique `trial_end` (plus nécessaire)

## Backfill DB
SWITAA (`tenant-1771276916432`) avait `current_period_end: NULL` en DEV.
Backfill appliqué :
```sql
UPDATE billing_subscriptions
SET current_period_end = (SELECT created_at + interval '14 days' FROM tenants WHERE id = billing_subscriptions.tenant_id)
WHERE status = 'trialing' AND current_period_end IS NULL
```
Résultat : `current_period_end = 2026-03-02T21:21:56.445Z`
