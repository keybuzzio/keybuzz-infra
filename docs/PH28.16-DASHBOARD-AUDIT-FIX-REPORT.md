### Analyse du Code

**Problème identifié:**

Dans `src/features/dashboard/api/dashboard.service.ts`, si l'API `/stats/dashboard` échoue, le code tombait en fallback vers des **données démo**:

```typescript
// AVANT (problématique)
if (apiResult) {
  return apiResult;
}

// Fallback to demo data if API is unavailable
return {
  kpis: computeKpisFromDemo(apiFormatData),
  source: 'demo',  // ← BUG: données démo pour user authentifié
};
```

**Comportement interdit:**
- User authentifié + API fail → Affichait "Données démo" au lieu d'un message d'erreur

---

## Fix Appliqué

### Fichiers Modifiés

| Fichier | Changement |
|---------|------------|
| `src/features/dashboard/api/dashboard.service.ts` | Remplacé fallback `demo` par `error` |
| `src/features/dashboard/types.ts` | Mis à jour type `source: 'api' \| 'error'` |
| `app/dashboard/page.tsx` | Badge et banner adaptés pour état `error` |

### Code Après Fix

```typescript
// APRÈS (corrigé)
if (apiResult) {
  return apiResult;
}

// PH28.16: API unavailable - return error state, NEVER demo data
return {
  kpis: {
    openConversations: 0,
    pendingConversations: 0,
    // ... all zeros
  },
  source: 'error',  // ← Explicit error state
};
```

### UI Error State

Quand `source === 'error'`:
- Badge: "API indisponible" (couleur amber)
- Banner: "API indisponible - Impossible de charger les statistiques. Cliquez sur Actualiser pour réessayer."
