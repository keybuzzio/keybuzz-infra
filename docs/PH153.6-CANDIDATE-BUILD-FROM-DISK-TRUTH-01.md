## 3. ANALYSE DE L'ECHEC

### Cause racine

Le fichier `app/inbox/InboxTripane.tsx` a ete remplace par la version de `ph152.6-client-parity`, qui importe `ConversationActionBar` (ligne 35). Ce composant :

- Est AJOUTE dans ph152.6 (n'existe pas dans release)
- A **0 references** dans l'image compilee v3.5.63
- N'a donc **pas ete ajoute au moment du build** de v3.5.63

### La modification de InboxTripane.tsx dans ph152.6 est POSTERIEURE au build v3.5.63

Preuve par analyse croisee des trois versions :


| Version                 | Lignes | EscalationPanel     | ConversationActionBar    |
| ----------------------- | ------ | ------------------- | ------------------------ |
| release/client-v3.5.220 | 1903   | **OUI** (2 imports) | **NON**                  |
| ph152.6-client-parity   | 1903   | OUI                 | **OUI** (ajout ligne 35) |
| .bak (disque)           | 1656   | NON                 | NON                      |
| v3.5.63 compilee        | -      | **OUI** (16 chunks) | **NON** (0 chunks)       |


- v3.5.63 A EscalationPanel → matche release, PAS le .bak
- v3.5.63 N'A PAS ConversationActionBar → matche release, PAS ph152.6
- **Conclusion** : InboxTripane.tsx de v3.5.63 = **version release** (non modifiee)

### Implication pour ph153.5-disk-truth

La strategie PH153.5 a applique les 63 fichiers modifies de ph152.6 en bloc. Mais au moins un fichier (`InboxTripane.tsx`) n'avait PAS ete modifie au moment du build. **Tous les 63 fichiers ne sont pas egaux** — certaines modifications de ph152.6 sont anterieures au build, d'autres posterieures.
