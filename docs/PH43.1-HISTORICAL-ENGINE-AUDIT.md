## 2. STRUCTURE ACTUELLE DU PROMPT

```
[1]   Base prompt (regles generales)                  ~150 tokens
[1.5] SAV POLICY ENGINE (PH41)                        ~700-800 tokens
[2]   CONTEXTE COMMANDE (si order_ref)                ~200-300 tokens
[3]   REGLES SPECIFIQUES DU VENDEUR (si ai_rules)     variable
[4]   CONTEXTE FOURNISSEUR (si supplier_case)         ~100 tokens
[5]   Instructions finales (contextType + format)     ~50 tokens
```

Total system prompt actuel : **~1300-1500 tokens**

## 3. POINT D'INJECTION

Inserer le bloc HISTORICAL RESOLUTION entre [1.5] et [2] :

```
[1]   Base prompt
[1.5] SAV POLICY ENGINE (PH41)
[1.6] HISTORICAL RESOLUTION PATTERNS (PH43) ← NOUVEAU
[2]   CONTEXTE COMMANDE
[3]   REGLES VENDEUR
[4]   CONTEXTE FOURNISSEUR
[5]   Instructions finales
```

**Justification** : Les patterns historiques doivent etre lus APRES la politique SAV
(qui definit les regles) et AVANT le contexte commande (qui donne les faits).
L'IA lit la politique, puis les precedents, puis les faits = raisonnement optimal.
