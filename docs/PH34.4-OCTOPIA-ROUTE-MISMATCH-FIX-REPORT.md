# PH34.4 — Fix Octopia Route Mismatch + Header Fix

## Probleme 1 : Route mismatch (BFF → Backend)

### Cause racine
Les routes BFF Next.js (sous `app/api/octopia/`) appelaient le backend avec le prefix `/octopia/*`,
mais le backend Fastify enregistre toutes les routes Octopia sous `/marketplaces/octopia/*`.

### Routes corrigees (5/6)

| BFF Route (client) | Avant (backend call) | Apres (backend call) |
|---|---|---|
| `/api/octopia/test` | `/octopia/test` | `/marketplaces/octopia/test` |
| `/api/octopia/connect` | `/octopia/connect` | `/marketplaces/octopia/connect` |
| `/api/octopia/disconnect` | `/octopia/disconnect` | `/marketplaces/octopia/disconnect` |
| `/api/octopia/status` | `/octopia/status` | `/marketplaces/octopia/status` |
| `/api/octopia/sync/run` | `/octopia/sync` | `/marketplaces/octopia/sync/run` |
| `/api/octopia/config` | `/marketplaces/octopia/config` | *Inchange (deja correct)* |

### Fichiers modifies (client)
- `keybuzz-client/app/api/octopia/test/route.ts`
- `keybuzz-client/app/api/octopia/connect/route.ts`
- `keybuzz-client/app/api/octopia/disconnect/route.ts`
- `keybuzz-client/app/api/octopia/status/route.ts`

## Probleme 2 : Header Octopia API change (403 Forbidden)

### Cause racine
L'API Octopia a change le nom du header d'identification vendeur.
L'ancien header `X-Seller-Id` retourne systematiquement **403 Forbidden**.
Le nouveau header `sellerId` (camelCase, sans tiret) retourne **200 OK**.

### Diagnostic (tests effectues dans le pod API DEV)

| Header | Status | Resultat |
|--------|--------|----------|
| `X-Seller-Id: 273340` | **403** | Forbidden |
| `x-seller-id: 273340` | **403** | Forbidden |
| `x-sellerid: 273340` | **403** | Forbidden |
| **`sellerId: 273340`** | **200** | Commandes reelles recues |

Token OAuth OK (200), JWT valide avec `azp: KeyBuzz` et scopes ORDERS_GET.
Le 403 venait exclusivement du header, pas d'un probleme de delegation.

### Fichier modifie (API)
- `keybuzz-api/src/modules/marketplaces/octopia/octopiaClient.ts` (ligne 91)

Avant: `'X-Seller-Id': sellerId`
Apres: `'sellerId': sellerId`

### Preuve test DEV
```

orders_status: 200
orders_count: 1
first_order: 2602051628MO4FD Cancelled
TEST: PASS
```
