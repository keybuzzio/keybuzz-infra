## Resume Executif

Deux bugs critiques empechaient le pipeline Autopilot de fonctionner en conditions reelles :

1. **PLAN_MODELS case sensitivity** : La table `tenants` stocke les plans en MAJUSCULES (`AUTOPILOT`, `PRO`), mais `PLAN_MODELS` n'avait que des cles minuscules. Resultat : tous les tenants AUTOPILOT et PRO (sauf `ecomlg-001` par chance) utilisaient le modele LLM le moins cher (`kbz-cheap`) au lieu de leur modele assigne.

2. **LiteLLM modele Anthropic deprecie** : Le fallback `anthropic/claude-3-5-haiku-20241022` du groupe `kbz-cheap` n'existe plus chez Anthropic (erreur 404). Quand LiteLLM routait vers ce fallback, l'appel echouait completement, retournant confiance 0 et declenchant une escalade silencieuse.

**Impact avant fix** : 100% des conversations Autopilot retournaient confiance 0 → escalade silencieuse → aucune reponse client generee.

**Impact apres fix** : confiance 0.75-0.85, reponses contextuelles riches avec tracking integre.

- Normalisation du plan en minuscules dans `getModelForPlan()` :
```typescript
export function getModelForPlan(plan: string): string {
  const normalized = plan.toLowerCase();
  return PLAN_MODELS[normalized] || PLAN_MODELS['starter'];
}
```
