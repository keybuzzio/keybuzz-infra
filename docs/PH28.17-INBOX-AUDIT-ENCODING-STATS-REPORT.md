# PH28.17 - Inbox Audit & Fix (Encoding + Stats) Report

**Date:** 2026-02-03
**Status:** ✅ COMPLETED
**Environments:** DEV + PROD

---

## Objectifs

1. Supprimer les caractères bizarres (encodage) dans les compteurs Inbox
2. Rendre l'état "Stats API indisponible" propre
3. Ne jamais afficher "Données démo" pour un user authentifié

---

## Problèmes Identifiés

### 1. Encodage Cassé

**Symptôme:** Les compteurs affichaient `â€"` au lieu de `-`

**Cause:** Le fichier `InboxTripane.tsx` contenait des bytes UTF-8 corrompus:
- Bytes: `c3a2 e282ac e2809d` (séquence UTF-8 pour "â€"")
- Origine: Character "—" (em-dash) mal encodé lors d'un copier/coller

**Preuve hexadécimale (avant fix):**
```
00000050: 6179 2d34 3030 223e c3a2 e282 ace2 809d  ay-400">........
```

### 2. Badge "Données démo"

**Symptôme:** Le badge affichait "Données démo" (jaune) quand la source n'était pas API

**Cause:** La logique du badge utilisait `"Données démo"` comme fallback
```jsx
// AVANT (problématique)
{dataSource === "api" ? "Données API" : "Données démo"}
```

**Règle violée:** Un utilisateur authentifié ne doit jamais voir "démo"

---

## Fixes Appliqués

### Fix 1: Encodage

Remplacement des bytes corrompus par un simple tiret `-`:

```python
# Bytes corrompus
bad_bytes = bytes([0xc3, 0xa2, 0xe2, 0x82, 0xac, 0xe2, 0x80, 0x9d])
# Remplacement
good_bytes = b'-'
```

**4 occurrences corrigées** dans `InboxTripane.tsx`

### Fix 2: Badge

```jsx
// APRÈS (corrigé)
{dataSource === "api" ? "Données API" : "API indisponible"}
```

Couleurs également mises à jour: jaune → amber pour l'état erreur.
