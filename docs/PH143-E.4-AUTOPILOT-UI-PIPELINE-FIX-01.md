## Resume Executif

Correction du pipeline UI complet de l'Autopilot : le drawer IA s'ouvre automatiquement quand un draft autopilot est disponible, le bouton "Brouillon IA disponible" fetche le draft en priorite, et les actions (Valider / Modifier / Ignorer) sont connectees au backend via `/autopilot/draft/consume`.

---

## Probleme Identifie

Le composant `AISuggestionSlideOver` (drawer IA) avait deja tout le code de rendu pour les drafts autopilot :
- Affichage `activeDraft` avec confiance, escalation status
- Boutons Valider / Modifier / Ignorer
- Auto-open quand `initialDraft` + `autoOpen` sont fournis

**MAIS** le composant parent `InboxTripane.tsx` ne passait aucune de ces props :
- Pas de `initialDraft` (toujours `undefined`)
- Pas de `autoOpen` (toujours `undefined`)
- Pas de `onDirectSend` (envoi direct impossible)
- Aucun fetch de `/api/autopilot/draft` au chargement de la conversation

Le bouton "Ouvrir l'assistant" dans le `AISuggestionsPanel` scrollait et cliquait le bouton "Aide IA", ouvrant le mode suggestion au lieu du draft autopilot.

## Fichiers Modifies

### 1. `src/features/ai-ui/index.ts`
- Export du type `AutopilotDraft` (necessaire pour le typage dans InboxTripane)

### 2. `app/inbox/InboxTripane.tsx` (3 ajouts)

#### A. State autopilot draft
```typescript
const [autopilotDraft, setAutopilotDraft] = useState<AutopilotDraft | null>(null);
const [autopilotAutoOpen, setAutopilotAutoOpen] = useState(false);
```

#### B. useEffect fetch draft
A chaque changement de conversation (`selectedId`), un fetch vers `/api/autopilot/draft` est effectue :
- Si `hasDraft: true` â†’ `setAutopilotDraft(...)` + `setAutopilotAutoOpen(true)`
- Sinon â†’ reset a `null`

#### C. Props drawer
```typescript
<AISuggestionSlideOver
  ...
  initialDraft={autopilotDraft}
  autoOpen={autopilotAutoOpen}
  onDirectSend={async (text) => { await sendReply(...); }}
/>
```

#### D. onOpenReplyAssistant
Le handler du bouton "Ouvrir l'assistant" dans le `AISuggestionsPanel` fetche d'abord le draft. Si un draft existe, il l'affiche directement. Sinon, fallback vers le mode suggestion classique.
