## Anomalies Detectees

### ANO-01 : Casse de plan inconsistante (HAUTE)

**Probleme** : `ecomlg-001` a `plan='pro'` (minuscule) dans la DB. Les autres tenants ont `plan='PRO'` (majuscule). Le type TypeScript `PlanType` est `'STARTER' | 'PRO' | 'AUTOPILOT'`.

**Risque** : Si `getPlanCapabilities(plan)` est appele avec `'pro'` (minuscule), la lookup dans `PLAN_CAPABILITIES['pro']` retourne `undefined`. Le tenant pilote pourrait ne pas beneficier des restrictions/capabilities PRO.

**Verification necessaire** : Inspecter `useEntitlement()` et `useCurrentPlan()` pour voir si la normalisation uppercase est faite.

### ANO-02 : STARTER tenant sans wallet (MOYENNE)

**Probleme** : Le tenant Essai n'a aucun enregistrement dans `ai_actions_wallet`. Sans wallet, les features IA necessitant des KBActions ne fonctionneront pas.

**Cause probable** : Tenant cree avant l'ajout du systeme de wallet. Pas de migration backfill.

### ANO-03 : Addon gating non testable en trial (MOYENNE)

**Probleme** : Tous les 7 tenants AUTOPILOT sont en `trialing`. Le trial deverrouille toutes les features. Impossible de tester le gating addon (BILL-02) sans un tenant AUTOPILOT post-trial.

### ANO-04 : 0 AI_AUTO_ESCALATED malgre 5 escalations

**Probleme** : Le tenant AUTOPILOT a 5 conversations avec status "Escalade" mais 0 evenement `AI_AUTO_ESCALATED` dans `ai_action_log`. Les escalations sont probablement manuelles (via bouton "Escalader" dans l'inbox).

**Cause** : Le mode `supervised` (qui est le mode actif) ne declenche pas d'auto-escalade. Il faudrait le mode `autonomous` actif + un message contenant une fausse promesse IA pour declencher `AI_AUTO_ESCALATED`.
