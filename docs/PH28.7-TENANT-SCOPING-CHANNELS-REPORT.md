## Audit Technique

### Endpoint API

**URL**: `/api/amazon/status?tenant_id={tenantId}`

**Flow**:
1. Client appelle `/api/amazon/status?tenant_id=xxx`
2. BFF Next.js extrait `tenant_id` du query param
3. BFF appelle backend avec header `X-Tenant-Id`
4. Backend filtre par `tenantId` dans la requête DB

### Code Backend

```typescript
// src/modules/marketplaces/amazon/amazon.routes.ts
server.get("/api/v1/marketplaces/amazon/status", { preHandler: authenticate }, async (request, reply) => {
  const user = request.user;
  
  if (!user || !user.tenantId) {
    return reply.status(401).send({ error: "Unauthorized" });
  }

  const connection = await prisma.marketplaceConnection.findFirst({
    where: {
      tenantId: user.tenantId,  // ✅ Filtrage strict par tenant
      type: MarketplaceType.AMAZON,
    },
  });

  // ...
});
```

## Corrections Apportées

**Aucune correction n'a été nécessaire.**

L'isolation tenant pour les canaux/connexions Amazon était déjà correctement implémentée :
- Frontend passe `tenant_id` dans chaque requête
- Backend filtre strictement par `tenantId` de la session
- DB stocke toutes les connexions avec `tenantId` obligatoire
