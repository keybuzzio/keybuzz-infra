## Bug Identifié

### Symptôme
"Tenant introuvable. Veuillez vous reconnecter." affiché même pour un user authentifié avec tenant.

### Cause Racine
Le Journal IA utilisait uniquement `getLastTenant()` / `getCurrentTenantId()` de localStorage, qui n'était pas toujours peuplé après signup.

```typescript
// AVANT - Dépendance unique sur localStorage
const lastTenant = getLastTenant();
let rawTenantId = lastTenant.canonicalId || getCurrentTenantId();
// Si localStorage vide → erreur "Tenant introuvable"
```

---

## Fix Appliqué

### Ajout TenantProvider comme source primaire

```typescript
// APRÈS (PH28.22) - TenantProvider + localStorage fallback
import { useTenant } from "@/src/features/tenant/TenantProvider";

const { currentTenantId: providerTenantId } = useTenant();

useEffect(() => {
  // Priorité 1: TenantProvider
  if (providerTenantId) {
    console.log('[AIJournal] Using TenantProvider:', providerTenantId);
    setTenantId(providerTenantId);
    return;
  }
  
  // Priorité 2: localStorage (fallback)
  const lastTenant = getLastTenant();
  let rawTenantId = lastTenant.canonicalId || getCurrentTenantId();
  // ...
}, [providerTenantId]);
```
