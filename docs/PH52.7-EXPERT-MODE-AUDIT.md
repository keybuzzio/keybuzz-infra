# PH52.7 — Expert Mode Audit

> Date : 9 mars 2026
> Phase : PH52.7.1

## Etat avant PH52.7

### Table `tenant_ai_learning_settings`
```sql
tenant_id TEXT PRIMARY KEY
learning_collect_enabled BOOLEAN DEFAULT true
learning_apply_enabled BOOLEAN DEFAULT false
created_at TIMESTAMPTZ
updated_at TIMESTAMPTZ
```

Pas de colonne pour distinguer Expert d'Adaptatif.

### Mapping UI -> Backend (PH52.6)

| Mode UI | collectEnabled | applyEnabled | Resultat backend |
|---------|---------------|-------------|-----------------|
| Standard | true | false | standard |
| Adaptatif | true | true | adaptive |
| Expert | true | true | **= adaptive (identique)** |

`modeFromControl()` ne retournait jamais `'expert'`.

### Fichiers impactes

| Fichier | Impact |
|---------|--------|
| `learningControlService.ts` | Interface + DB query + update |
| `ai-policy-debug-routes.ts` | GET/POST endpoints + PolicyEffectiveResponse |
| `ai-assist-routes.ts` | decisionContext.learningControl |
| `LearningControlSection.tsx` | modeFromControl + controlFromMode + state |
| `route.ts` (BFF) | Proxy pass-through (aucun changement) |

### Conclusion

Ajouter `learning_advanced_ui_enabled BOOLEAN DEFAULT false` dans la table, propager le champ dans toute la chaine.
