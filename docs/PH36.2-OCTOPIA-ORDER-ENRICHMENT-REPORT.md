## Mapping Octopia → KeyBuzz

| Champ KeyBuzz | Source Octopia (raw_data) |
|---------------|--------------------------|
| `customerPhone` | `lines[0].shippingAddress.phone` ou `billingAddress.phone` |
| `marketplaceStatus.label` | Mapping `status` → {Creee, Acceptee, Expediee, En transit, Livree, Annulee, Remboursee} |
| `marketplaceStatus.step` | 1-5 (progres) ou negatif (annule/rembourse) |
| `timeline` | `purchasedAt`, `createdAt`, `parcels[].carrierName`, `updatedAt` |
| `financialSummary.sale` | `totalPrice.sellingPrice` |
| `financialSummary.commission` | `-lines[].offerPrice.commission.amountWithVat` |
| `financialSummary.serviceFees` | `-serviceFees[].amount` |
| `financialSummary.refunds` | 0 (pas de champ refund detecte dans raw_data) |
| `financialSummary.total` | sale - commission - serviceFees |
