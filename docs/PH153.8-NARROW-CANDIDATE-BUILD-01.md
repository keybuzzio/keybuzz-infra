## 4. ANALYSE DE L'ECHEC

### Cause racine : dependance de type implicite

`ProfileTab.tsx` (classe A) utilise `profile.siret` a la ligne 41. Ce champ `siret` doit etre defini dans le type `BusinessProfile`, qui reside dans `app/settings/types.ts` — un fichier classe **C** (preuve insuffisante).

La version release de `types.ts` NE CONTIENT PAS le champ `siret`. La version ph152.6 l'ajoute. Sans cette modification de types, ProfileTab.tsx ne compile pas.

### Decouverte : les fichiers A et C sont lies par des dependances de types

```
ProfileTab.tsx (A) --imports--> BusinessProfile (from types.ts, C)
                   --uses-->   profile.siret (added in ph152.6 types.ts)
                               profile.contactEmail (added in ph152.6 types.ts)
```

Le tri PH153.7 a prouve que ProfileTab.tsx est modifie (icons Building2/MapPin dans le compile). Mais il n'a pas pu prouver types.ts (const/type names minifies). Or types.ts est une **dependance implicite** de ProfileTab.tsx.

### Implication fondamentale

L'approche "A uniquement" est **insuffisante**. Certains fichiers C sont des dependances structurelles (types, interfaces, exports) des fichiers A. Un fichier A prouve ne peut compiler sans ses dependances C.

**Schemas de dependance identifies :**


| Fichier A                                | Dependance C probable                           |
| ---------------------------------------- | ----------------------------------------------- |
| `app/settings/components/ProfileTab.tsx` | `app/settings/types.ts` (BusinessProfile.siret) |
| Possiblement d'autres                    | `app/settings/constants.ts`, services, etc.     |
