# PH151.3 — Priority UX Consolidation

> **Date** : 14 avril 2026
> **Environnement** : DEV uniquement
> **Image** : `ghcr.io/keybuzzio/keybuzz-client:v3.5.64-ph151.3-priority-clean-dev`
> **Rollback** : `v3.5.63-ph151.2-case-summary-clean-dev`

---

## Objectif

Clarifier et simplifier le système de priorité dans l'Inbox sans ajouter de complexité, créer de doublons ou casser la logique existante. La priorité doit être **évidente d'un coup d'œil**.

## Problèmes identifiés (audit)


| Problème                                                         | Impact                                       |
| ---------------------------------------------------------------- | -------------------------------------------- |
| `PriorityBadge` (label + icône) prenait trop de place en largeur | Encombrement visuel sur chaque ligne         |
| `UnreadDot` (point bleu) redondant avec la logique de priorité   | Double signal visuel pour la même info       |
| Badge "Assigné à moi" trop long                                  | Consommation d'espace inutile                |
| Badge `supplierCaseStatus` dans la liste                         | Info secondaire polluant la ligne principale |
| Bloc "Prioritaires d'abord" trop verbeux                         | Stats difficilement lisibles                 |


## Changements appliqués

### 1. PriorityDot remplace PriorityBadge

- **Avant** : `<PriorityBadge level="high" label="Urgent — SLA dépassé" />` (label + icône, ~180px)
- **Après** : `<PriorityDot level="high" />` (point coloré 8px, position 1ère dans la ligne)
- Logique inchangée (`getConversationPriority()` dans `conversationPriority.ts`)

### 2. Suppression UnreadDot

- Le point bleu "non lu" est supprimé — la priorité couvre déjà ce cas (un message non lu non assigné = priorité haute)

### 3. Label "Assigné à moi" → "À moi"

- Raccourci pour gagner de l'espace sans perte de sens

### 4. Badge supplierCaseStatus retiré de la liste

- Info SAV fournisseur disponible dans le panneau détail

### 5. Bloc Priority Stats compact

- **Avant** : 3 lignes avec texte descriptif + barres
- **Après** : 1 ligne avec toggle + compteurs inline (point rouge + nombre, point ambre + nombre)
