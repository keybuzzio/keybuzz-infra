> Date : 8 mars 2026

## 1. Briques existantes analysees

### PH41 — SAV Policy Engine (`sav-policy.ts`)
- **12 scenarios** : delivery_delay, delivered_not_received, refund_request, return_request, defective_product, damaged_product, wrong_product, warranty_request, aggressive_customer, cancellation_request, invoice_request, unknown
- **Structure** : `ScenarioRule { scenario, label, priority, promptBlock }`
- **Prompt** : bloc statique par scenario (regles textuelles injectees dans le prompt)
- **Limite** : les regles sont **narratives** (texte libre), pas structurees en arbre de decision

### PH41 — SAV Classifier (`savClassifier.ts`)
- **Heuristique** : keywords + phrases + regex par scenario
- **Sortie** : `{ scenario: SavScenario, confidence: number, matchedPatterns: string[] }`
- **Confidence** : score normalise sur maxPossible=50 (0.0 a 1.0)
- **Limite** : la confiance mesure la certitude de CLASSIFICATION, pas la certitude de DECISION

### PH43 — Historical Resolution Engine
- **Dataset** : `sav-fused-dataset.json` (cas historiques)
- **Sortie** : `{ cases: SearchResult[], antiPatterns: AntiPatternSignal[], promptBlock: string }`
- **8 anti-patterns** avec severity warning/critical
- **Limite** : les anti-patterns sont des signaux negatifs, pas un arbre de decision positif

### PH44 — Tenant Policy
- **Champs** : refundThreshold, requirePhotos, supplierFirst, replaceFirst, diagnosticRequired, maxRefundWithoutReturn, toneStyle
- **Limite** : politique tenant statique, pas de logique conditionnelle

### PH44.5 — Observability
- **decisionContext** : objet JSON persiste dans `ai_actions_ledger`
- **Endpoint debug** : `/ai/policy/effective`
- **Point d'extension** : peut accueillir les nouveaux champs PH45

### PH44.7 — Order Context Enrichment
- **Nouveaux champs** : marketplaceId, marketplaceLabel, deliveryWindow, itemCount, items[].asin/sku
- **Impact PH45** : la fenetre de livraison est un signal cle pour delivery_delay / delivered_not_received

## 2. Ecart identifie

| Probleme | Cause | Impact |
|---|---|---|
| L'IA repond avec le meme niveau d'assurance quel que soit le contexte | Pas de confidence gate sur la DECISION (seulement sur la CLASSIFICATION) | Reponses prematurees |
| L'IA peut proposer un remboursement meme sans preuve | Le prompt dit "ne pas rembourser" mais pas de verification structuree | Risque financier |
| L'IA ne detecte pas les infos manquantes | Pas de liste de signaux requis par scenario | Questions non posees |
| Les anti-patterns sont negatifs | Ils disent ce qu'il ne faut PAS faire, pas ce qu'il FAUT faire | Manque de guidance positive |

## 3. Point d'injection recommande

### Ordre actuel du prompt :
```
1. Base prompt (regles generales)
2. SAV Policy block (PH41 — regles scenario)
3. Tenant Policy block (PH44 — overrides tenant)
4. Historical patterns block (PH43 — cas passes + anti-patterns)
5. Order Context block (PH44.7 — donnees commande)
6. Tenant Rules (ai_rules)
7. Supplier Context
```

### Insertion PH45 :
```
1. Base prompt
2. SAV Policy block (PH41)
3. Tenant Policy block (PH44)
4. Historical patterns block (PH43)
5. >>> SAV DECISION TREE (PH45) <<< — NOUVEAU
6. Order Context block (PH44.7)
7. Tenant Rules
8. Supplier Context
```

**Justification** : le decision tree doit etre injecte APRES les regles et l'historique (pour les synthetiser) et AVANT le contexte commande (pour que le LLM applique le cadre au cas concret).

## 4. Architecture cible PH45

```
classifySavScenario(text) → scenario + classificationConfidence
                                ↓
evaluateDecisionTree(scenario, availableSignals) →
  {
    scenario,
    decisionConfidence: high | medium | low,
    missingSignals: [...],
    allowedActions: [...],
    forbiddenActions: [...],
    nextBestStep: string,
    promptBlock: string
  }
                                ↓
buildSystemPrompt(..., decisionTreeBlock)
```

### Signaux disponibles (inputs du decision tree) :
- `hasOrderContext` — commande trouvee en DB
- `hasDeliveryWindow` — fenetre de livraison connue
- `hasTracking` — numero de suivi disponible
- `hasDeliveryStatus` — statut livraison connu
- `orderStatus` — Shipped/Unshipped/Cancelled
- `hasPhotos` — pieces jointes dans la conversation
- `hasMultipleMessages` — historique d'echanges
- `orderValue` — montant commande (pour seuil refund)
- `tenantRequiresPhotos` — politique tenant
- `tenantSupplierFirst` — politique fournisseur d'abord
- `tenantRefundThreshold` — seuil remboursement tenant
- `historicalCasesFound` — cas historiques trouves
- `antiPatternsTriggered` — anti-patterns actifs
- `daysSincePurchase` — jours depuis l'achat
- `deliveryWindowPassed` — fenetre de livraison depassee
