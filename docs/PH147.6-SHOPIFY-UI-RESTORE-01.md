## CAUSE RACINE

Le code client (`app/channels/page.tsx`) contenait tout le wiring Shopify :

- imports (`getShopifyStatus`, `connectShopify`, `disconnectShopify`)
- state (`shopifyStatus`, `showShopifyModal`, `shopifyDomain`, `shopifyConnecting`)
- handlers (`handleShopifyConnect`, `handleShopifyDisconnect`)
- rendu conditionnel (bouton "Connecter Shopify", modal de connexion, badge status)
- logo (`/marketplaces/shopify.svg` â€” 714 octets, present)

**Mais l'API backend (`v3.5.55-ph147.4-source-of-truth-dev`) ne contient AUCUNE reference a Shopify** :

- `grep -rin shopify /opt/keybuzz/keybuzz-api/src/` = 0 resultats
- `/shopify/status` retourne 404
- `/channels/catalog` retourne uniquement `amazon, octopia, fnac, darty`

La page Channels affiche les canaux depuis deux sources :

1. `fetchTenantChannels()` â†’ canaux deja ajoutes au tenant (aucun Shopify)
2. `fetchChannelsCatalog()` â†’ catalogue des plateformes disponibles (pas de Shopify dans l'API)

**Consequence** : Shopify n'apparaissait jamais dans l'interface, ni dans les canaux actifs, ni dans le modal d'ajout.
