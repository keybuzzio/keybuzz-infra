## Cause Racine

**Le fichier `build-metadata.ts` était généré avec la version correcte (`1.0.0-ph29.1`) APRÈS le build Docker initial.**

Timeline:
1. 19:35 - Build Docker avec version 0.2.85 (ancienne)
2. 21:00 - package.json mis à jour vers 1.0.0-ph29.1
3. L'image `ph29.1-onboarding` contenait donc l'ancienne version hardcodée

---

## Fix Appliqué

1. Rebuild `npm run build` → génère `build-metadata.ts` avec `1.0.0-ph29.1`
2. Docker build avec nouveau tag `ph29.0-dev-fix-2026-02-03`
3. Push et déploiement DEV
