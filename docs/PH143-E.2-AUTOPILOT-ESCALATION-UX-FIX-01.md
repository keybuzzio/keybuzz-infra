## 1. Resume executif

Correction du comportement critique ou l'IA escaladait une conversation sans informer le client.
Deux chemins d'escalation silencieuse identifies et corriges :

| Path | Probleme | Fix |
|------|----------|-----|
| Path 1 (low confidence, no draft) | escaladeConversation() appelee sans message client | Genere ESCALATION_FALLBACK_REPLY, passe par pipeline draft (safe_mode) ou envoi direct (autonome) |
| Path 2 (case 'escalate' dans switch) | Seul le statut conversation est change | Envoie le fallback reply via executeReply() AVANT l'escalade |

## 2. Fichiers modifies

### `src/modules/autopilot/engine.ts`
- Ajout constante `ESCALATION_FALLBACK_REPLY` avec message d'accuse reception client
- `SAFE_MODE_BLOCKED_ACTIONS` : ajout `'escalate'` (empeche l'escalade silencieuse en safe_mode)
- Path 1 (ligne ~273) : en safe_mode genere `ESCALATION_DRAFT` pour validation; en autonome envoie reply + escalade
- Path 2 (case 'escalate') : envoie toujours le fallback reply avant d'escalader

### `src/modules/autopilot/routes.ts`

- `/autopilot/draft/consume` POST : query inclut `ESCALATION_DRAFT%` pour pouvoir consommer les drafts d'escalation

- Le code d'escalation utilise le fallback reply dans les 2 paths corriges
