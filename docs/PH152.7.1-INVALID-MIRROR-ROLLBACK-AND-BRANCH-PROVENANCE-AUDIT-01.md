## 5. Analyse — Où se trouve le code de v3.5.63 ?

### Fait établi

L'image a été construite depuis le **disque du bastion** le 13 avril à 21:25 UTC, pendant que la branche `release/client-v3.5.220` (commit `2adbd40`) était checkout. Le Dockerfile utilise `COPY app ./app`, `COPY src ./src`, `COPY public ./public` — il copie TOUT le contenu disque, y compris les fichiers non commités.

### Conséquence

Le code source de v3.5.63 est l'**union** de :

- Les fichiers trackés de `release/client-v3.5.220` @ `2adbd40` (407 fichiers)
- **PLUS** les fichiers non commités qui étaient sur le disque au moment du build

### Problème fondamental

Ces fichiers non commités n'ont jamais été capturés dans Git. Ils n'existent potentiellement que :

1. Sur le disque du bastion (s'ils n'ont pas été écrasés par les checkouts ultérieurs)
2. Dans le workspace local Windows (13 fichiers non commités identifiés)
3. Dans l'image Docker compilée (sous forme de JS webpack, pas de TypeScript)

### Routes manquantes entre `release/client-v3.5.220` et l'image

L'image contient 134 routes, la branche en contient 130. Les 4 routes manquantes :

- `api/shopify/status` — jamais commité dans aucune branche
- `api/shopify/connect` — jamais commité dans aucune branche
- `api/shopify/disconnect` — jamais commité dans aucune branche
- `api/space-invites/resolve` — existe dans `rebuild/ph143-client` mais pas dans `release/client-v3.5.220`

### Autres fichiers manquants potentiels (non-routes)

Le disque du bastion a été modifié par de multiples checkouts depuis le 13 avril. Les fichiers `src/` non commités qui étaient présents lors du build sont potentiellement perdus ou écrasés.
