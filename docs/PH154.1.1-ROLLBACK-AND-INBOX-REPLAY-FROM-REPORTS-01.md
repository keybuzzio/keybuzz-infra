## LOGIQUE DE CLASSIFICATION

### isAmazonAutoMessage() — conservative, multi-signaux

Exige au minimum **2 signaux forts concordants** parmi 10 possibles :


| #   | Signal                                            | Exemple                                     |
| --- | ------------------------------------------------- | ------------------------------------------- |
| 1   | `copyright` + `amazon`                            | "Copyright 2026 Amazon, Inc."               |
| 2   | `seller central`                                  | "rendez-vous sur Seller Central"            |
| 3   | `preferences de notification`                     | "Modifiez vos preferences de notifications" |
| 4   | `veuillez ne pas repondre` / `do not reply`       | "Veuillez ne pas repondre a ce message"     |
| 5   | `cet e-mail a ete envoye` / `this email was sent` | "Cet e-mail a ete envoye par Amazon"        |
| 6   | `amazon eu s.` + `r.l`                            | "Amazon EU S.a r.l."                        |
| 7   | `amazon.com, inc` / `amazon services`             | "Amazon.com, Inc."                          |
| 8   | `modifiez vos preferences` / `modify your`        | "Modifiez vos preferences"                  |
| 9   | `capital social` + `eur`                          | "Capital social : 165 833 EUR"              |
| 10  | `spc-eu`                                          | "SPC-EUAmazon-200553108..."                 |


### Pourquoi >=2 : eviter les faux positifs

Un message client comme "JE DEMANDE LE REMBOURSEMENT IMMEDIAT" ne matchera **aucun** de ces signaux. Seuls les vrais messages systeme Amazon contiennent plusieurs de ces patterns simultanement.

Cas test critique : conversation `405-3091046-2681903` — le message client "JE DEMANDE LE REMBOURSEMENT" = 0 signaux → correctement classe comme CLIENT.
