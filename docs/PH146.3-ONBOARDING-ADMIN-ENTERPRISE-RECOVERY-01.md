## 2. Chaîne onboarding → user/admin → entreprise

### Flux complet

```
/register (page inscription)
  ├── Étape "company" : collecte companyName, siret, street, zipCode, city, country, companyPhone, supportEmail
  ├── Étape "user" : collecte firstName, lastName, phone, email
  └── Submit → POST /api/auth/create-signup (BFF)
        └── Proxy → POST /tenant-context/create-signup (API backend)
              ├── Crée user dans `users`
              ├── Crée tenant dans `tenants` (name = companyName)
              ├── Crée association dans `user_tenants` (role = owner)
              ├── Crée metadata dans `tenant_metadata` (tous les champs)
              └── Crée wallet dans `ai_actions_wallet`
```

## 3. Cause racine

Le type `BusinessProfile` dans `app/settings/types.ts` n'avait que 4 champs (`companyName`, `email`, `phone`, `returnAddress`), alors que l'API backend avait été enrichie avec 6 champs supplémentaires (`siret`, `street`, `zipCode`, `city`, `country`, `companyPhone`).

Le `ProfileTab.tsx` n'affichait que les 4 champs du type. Les données collectées pendant l'onboarding étaient **bien stockées en DB** mais **invisibles** dans les paramètres.

Le BFF (`app/api/tenant-context/profile/[tenantId]/route.ts`) est un passthrough pur — il n'a jamais filtré les champs. Le problème était exclusivement côté client.

---

## 4. Fix appliqué

### Fichiers modifiés (4 fichiers)


| Fichier                                  | Modification                                                                                 |
| ---------------------------------------- | -------------------------------------------------------------------------------------------- |
| `app/settings/types.ts`                  | Ajout des 6 champs manquants au type `BusinessProfile`                                       |
| `app/settings/constants.ts`              | Ajout des valeurs par défaut pour les nouveaux champs                                        |
| `app/settings/components/ProfileTab.tsx` | Ajout des champs UI : SIRET, Téléphone entreprise, Pays, Rue, Code postal, Ville             |
| `app/settings/page.tsx`                  | Mise à jour du chargement (GET) et de la sauvegarde (PUT) pour inclure les 6 nouveaux champs |


### Détail du ProfileTab enrichi

Deux sections structurées :

1. **Identité entreprise** : Nom de la société, SIRET, Email de contact, Téléphone principal, Téléphone entreprise, Pays (sélecteur)
2. **Adresse** : Rue, Code postal, Ville, Adresse de retour (textarea)
