## 4. CHAINE TECHNIQUE AUDITEE

```
Inbound message → API /inbound/routes.js (l.209, l.441)
  → evaluateAndExecute(conversationId, tenantId, 'inbound')
    → /modules/autopilot/engine.js
      → Load autopilot_settings (mode=supervised)
      → Verify plan, KBActions
      → Load conversation context + order context
      → LLM call via litellm_service
      → Score confidence
      → Guardrails: false promise detection
      → Generate draft → INSERT ai_action_log (status='planned')
      → Debit KBActions

Client: InboxTripane.tsx selectionne conversation
  → AutopilotDraftBanner mount
    → fetch /api/autopilot/draft?conversationId=xxx&tenantId=yyy
      → BFF proxy → API /autopilot/draft
        → SELECT FROM ai_action_log WHERE conversation_id = $1 AND status IN ('planned','skipped')
        → Return {hasDraft: true, draftText: "...", confidence: 0.75, ...}
    → Render banner avec texte, score, boutons
    → User clique "Valider et envoyer" ou "Modifier" ou "Ignorer"
```

## 5. CAUSE RACINE

**Cause racine unique : `AutopilotDraftBanner` n'a JAMAIS ete branche dans `InboxTripane.tsx` dans Git.**

Le composant existait en tant que fichier orphelin dans `src/features/inbox/components/AutopilotDraftBanner.tsx`. Il etait complet et fonctionnel (219 lignes, UI riche avec score de confiance, alertes escalade, boutons d'action), mais :

1. Il n'etait **pas importe** dans `InboxTripane.tsx`
2. Il n'etait **pas rendu** dans le JSX de l'inbox
3. Les routes BFF `/api/autopilot/draft` et `/api/autopilot/draft/consume` n'etaient **pas trackees** dans Git

**Ce n'est PAS une regression du reset Git PH152.** L'integration n'avait jamais ete finalisee dans le code source. Ni DEV ni PROD ne montraient le draft banner.

### Arbre de causes

1. **AutopilotDraftBanner.tsx** — fichier non tracke (`??` dans git status) — jamais commite
2. **app/api/autopilot/draft/route.ts** — fichier non tracke — jamais commite
3. **app/api/autopilot/draft/consume/route.ts** — fichier non tracke — jamais commite
4. **InboxTripane.tsx** — aucun import de AutopilotDraftBanner
5. **useCurrentPlan.tsx** — manquait `hasAgentKeybuzzAddon` (dependance build de AutopilotSection)
