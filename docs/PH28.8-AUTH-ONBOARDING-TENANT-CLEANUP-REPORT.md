## PART C - Fix Hard: No Tenant = No App

### Removed `tenant_test_dev` Fallbacks

**Files Modified**:
- `app/api/amazon/inbound-address/route.ts`
- `app/api/amazon/disconnect/route.ts`
- `app/api/amazon/oauth/start/route.ts`

**Before**:
```typescript
const tenantId = url.searchParams.get('tenant_id') || (session as any).tenantId || 'tenant_test_dev';
```

**After**:
```typescript
const tenantId = url.searchParams.get('tenant_id') || (session as any).tenantId;
if (!tenantId) {
  return NextResponse.json({ error: 'tenant_id required' }, { status: 400 });
}
```

**Behavior**: All API routes now return 400 if `tenantId` is absent - no silent fallback.

## PART D - Fix Signup "Email Exists"

### New Endpoint: `/api/v1/auth/check-email`

**Backend** (`keybuzz-backend/src/modules/auth/auth.routes.ts`):
```typescript
app.post("/api/v1/auth/check-email", async (request, reply) => {
  const { email } = request.body || {};
  const user = await prisma.user.findUnique({
    where: { email: email.toLowerCase() },
  });
  return {
    exists: !!user,
    hasTenants: !!user?.tenantId,
  };
});
```

**Client** (`app/api/auth/check-email/route.ts`):
- Proxies to backend
- Returns `{ exists: boolean, hasTenants: boolean }`

**Signup Flow** (`app/signup/page.tsx`):
- Added `'exists'` step
- Before sending code, calls `/api/auth/check-email`
- If user exists, shows "Ce compte existe déjà" with login CTA
