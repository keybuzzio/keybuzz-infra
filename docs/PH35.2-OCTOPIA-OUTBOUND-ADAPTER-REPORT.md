# PH35.2 — Octopia Outbound Adapter (DEV only)

## Objectif

Permettre l'envoi de messages depuis l'Inbox KeyBuzz vers Octopia via l'API Octopia reelle (`POST /messages`).

## Modifications

### 1. `determineProvider()` — `messages/routes.ts`

```typescript
function determineProvider(channel: string, targetAddress: string | null): string {
  if (channel === 'amazon') return 'spapi';
  if (channel === 'octopia') return 'octopia';   // <-- PH35.2
  if (channel === 'email' && targetAddress?.includes('@')) return 'smtp';
  return 'mock';
}
```

**Avant** : `channel === 'octopia'` tombait dans le fallback `mock` → message marque "delivered" sans envoi reel.
**Apres** : `provider='octopia'` → handler dedie dans le worker.

### 2. `outboundWorker.ts` — Handler Octopia

Nouveau bloc `if (delivery.provider === 'octopia')` dans `processDelivery()` :

1. Extrait `octopiaDiscussionId` depuis `conversation.thread_key` (`octopia:{id}`)
2. Recupere `octopiaCustomerId` et `octopiaSalesChannel` depuis les metadata du premier message inbound
3. **Fallback** : si metadata absentes (conversations importees avant PH35.2), fetch direct de la discussion via `GET /discussions/{id}`
4. Obtient un token OAuth via `OCTOPIA_CLIENT_ID` / `OCTOPIA_CLIENT_SECRET`
5. Envoie via `POST /messages` avec payload :
   ```json
   {
     "discussionId": "<octopiaDiscussionId>",
     "body": "<message>",
     "receivers": [{ "userId": "<customerId>", "userType": "Customer" }],
     "salesChannel": "<salesChannel>"
   }
   ```
6. Si HTTP 201 → `markDelivered()` avec trace complète (messageId Octopia, discussionId, etc.)
7. Si erreur → `markFailed()` avec message d'erreur detaille

### 3. `octopiaImport.service.ts` — Metadata enrichies

Les messages importes stockent maintenant `octopiaCustomerId` et `octopiaSalesChannel` dans leur metadata JSON, evitant un round-trip API lors de l'envoi.

### 4. Mock fallback supprime

`channel === 'octopia'` retourne `provider='octopia'` et non plus `'mock'`. Si le handler Octopia echoue, le delivery passe en `failed` avec erreur explicite — jamais de faux `delivered`.

## Variables d'environnement

Ajoutees au worker deployment :

| Variable | Valeur |
|---|---|
| `OCTOPIA_AUTH_URL` | `https://auth.octopia-io.net/auth/realms/maas/protocol/openid-connect/token` |
| `OCTOPIA_API_URL` | `https://api.octopia-io.net/seller/v2` |
| `OCTOPIA_CLIENT_ID` | (via secret) |
| `OCTOPIA_CLIENT_SECRET` | (via secret) |
