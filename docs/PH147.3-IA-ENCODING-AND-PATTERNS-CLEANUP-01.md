mais les patterns `validateDraft` ne matchaient pas correctement les caractères accentués.

## Note technique : `u2014` dans le JS compilé

Le compilateur TypeScript convertit le caractère em-dash `—` (U+2014) en séquence d'échappement 
`\u2014` dans le JS compilé. C'est un comportement standard et valide — au runtime, 
JavaScript interprète `\u2014` comme le caractère `—`. 
Cela apparaît comme `u2014` dans les sorties `grep` mais n'affecte pas le comportement.
