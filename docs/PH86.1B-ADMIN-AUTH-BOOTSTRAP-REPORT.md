## 1. Objectif

Permettre a l'administrateur de se connecter a Admin v2 avec un systeme d'authentification securise, sans hardcodage, avec secrets geres par Vault + ESO.

---

## 2. Architecture Auth

### Flux d'authentification
```
Utilisateur → /login (email + password)
  → NextAuth CredentialsProvider
    → Compare email avec ADMIN_BOOTSTRAP_EMAIL (env var Vault)
    → bcryptjs.compare(password, ADMIN_BOOTSTRAP_PASSWORD_HASH)
    → Si valide : JWT token (8h) + session
    → Si invalide : erreur "Identifiants invalides"
```

### Chaine de confiance
```
Vault (kv/keybuzz/admin-v2/bootstrap)
  → ExternalSecret (keybuzz-admin-v2-bootstrap)
    → K8s Secret (Opaque, 2 keys)
      → Deployment env vars
        → NextAuth authorize() callback
          → bcryptjs.compare()
```

### RBAC
Roles definis dans `src/types/index.ts` :
- `super_admin` — acces complet
- `ops_admin` — operations
- `account_manager` — gestion clients
- `support_agent` — support
- `viewer` — lecture seule

Hierarchie dans `src/hooks/useAuth.ts` : `hasRole(requiredRole)` verifie que le niveau de l'utilisateur est >= au niveau requis.

## 6. Probleme resolu : bcryptjs standalone

### Probleme
Next.js standalone (`output: 'standalone'`) n'incluait pas `bcryptjs` dans le bundle. Le module etait absent de `node_modules/` dans l'image runner.

### Cause
Le traceur `@vercel/nft` de Next.js ne suit pas correctement les dependances utilisees dans les callbacks NextAuth (execution dynamique, pas de trace statique directe).

### Solution
1. `next.config.mjs` : ajout `experimental.outputFileTracingIncludes` pour forcer l'inclusion
2. `Dockerfile` : `COPY --from=builder /app/node_modules/bcryptjs ./node_modules/bcryptjs` (belt & suspenders)

### Verification
```
kubectl exec pod -- node -e "require('bcryptjs'); console.log('OK')"
→ bcryptjs loaded OK
→ Hash format valid: true
```
