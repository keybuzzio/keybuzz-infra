```
    → POST ${BACKEND_URL}/api/v1/orders/import-one   [keybuzz-api]
      → Si channel=amazon : Amazon SP-API (getOrder)

```

### Détection du channel

```typescript
// Dans import-one handler, AVANT l'appel Amazon :
const convChannelResult = await client.query(
  `SELECT channel FROM conversations WHERE tenant_id = $1 AND order_ref = $2 LIMIT 1`,
  [tenantId, orderRef]

if (channel === 'octopia') {
  return await importOctopiaOrderInline(client, tenantId, orderRef, reply);
}
```

## 3) Statut Octopia "Connecté"

### Avant

```
GET /octopia/status → { "connected": false, "status": "ERROR" }
```

Le status DB était `ERROR` car le background sync (orders) échouait :
`"response.orders is not iterable (cannot read property undefined)"`

L'API Octopia renvoie `{ items: [...], count: N }` pas `{ orders: [...], totalCount: N }`.

### Après

```
GET /octopia/status → { "connected": true, "status": "ERROR", "sellerId": "273340" }
```

Le fix considère le compte connecté si :
- `status = 'CONNECTED'` **OU**
- `status = 'ERROR'` ET `lastTestResult = 'OK'` (l'API fonctionne, seul le sync orders a échoué)

### Route BFF vérifiée

```
keybuzz-client/app/api/octopia/status/route.ts
→ fetch(`${API_URL}/octopia/status?tenantId=...`)
→ Route corrigée ✅
```
