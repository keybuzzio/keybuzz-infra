## 3. Problème identifié

### Cause racine
Le service client (`conversations.service.ts`) appelait l'API sans paramètre `limit`, donc recevait seulement **50 conversations** (limite par défaut du backend).

```typescript
// AVANT (api.ts)
conversations: (tenantId: string, searchQuery?: string) => 
  `${API_CONFIG.baseUrl}/messages/conversations?tenantId=${tenantId}...`

// APRÈS (api.ts)  
conversations: (tenantId: string, searchQuery?: string) => 
  `${API_CONFIG.baseUrl}/messages/conversations?tenantId=${tenantId}&limit=1000...`
```

---

## 4. Fix appliqué

### Fichier modifié
`/root/keybuzz-client/src/config/api.ts`

### Changement
Ajout de `&limit=1000` à l'endpoint conversations pour récupérer toutes les conversations.
