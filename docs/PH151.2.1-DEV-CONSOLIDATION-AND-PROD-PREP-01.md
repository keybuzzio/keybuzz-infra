### 5.1 Ordre de deploiement


| Etape | Service | Action              | Image source      | Image PROD cible                              |
| ----- | ------- | ------------------- | ----------------- | --------------------------------------------- |

| 3     | Backend | AUCUN               | Deja aligne       | —                                             |
| 4     | Workers | AUCUN               | Deja alignes      | —                                             |


**Justification de l'ordre** : l'API d'abord car les nouvelles routes BFF du client (PH143-E.9, PH147.4) requierent les endpoints API correspondants. Deployer le client avant l'API risquerait des erreurs 404/500 transitoires.

### 5.2 Commandes de deploiement

#### Etape 1 : API PROD

```bash
cd /opt/keybuzz/keybuzz-api
API_TAG="ghcr.io/keybuzzio/keybuzz-api:v3.5.55-ph147.4-source-of-truth-prod"
docker build --no-cache -t "$API_TAG" .
docker push "$API_TAG"
kubectl set image deploy/keybuzz-api keybuzz-api="$API_TAG" -n keybuzz-api-prod
kubectl rollout status deployment/keybuzz-api -n keybuzz-api-prod
```

#### Etape 2 : Client PROD

```bash
cd /opt/keybuzz/keybuzz-client
CLIENT_TAG="ghcr.io/keybuzzio/keybuzz-client:v3.5.75-ph151-step4.1-filters-collapse-prod"
docker build --no-cache \
  --build-arg NEXT_PUBLIC_API_URL=https://api.keybuzz.io \
  --build-arg NEXT_PUBLIC_API_BASE_URL=https://api.keybuzz.io \
  --build-arg NEXT_PUBLIC_APP_ENV=production \
  -t "$CLIENT_TAG" .
docker push "$CLIENT_TAG"
kubectl set image deploy/keybuzz-client keybuzz-client="$CLIENT_TAG" -n keybuzz-client-prod
kubectl rollout status deployment/keybuzz-client -n keybuzz-client-prod
```
