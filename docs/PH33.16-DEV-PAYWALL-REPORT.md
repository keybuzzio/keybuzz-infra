# PH33.16 — Paywall Trial + Grace 3 jours (DEV)

## Objectif
Implémenter un système complet de paywall :
- **Trial actif** → accès complet + bandeau "Essai actif J-x"
- **Past due** → grace 3 jours, puis lock total
- **Canceled/Expired** → lock immédiat
- **Active** → accès complet
- **Déblocage automatique** quand paiement OK

---

## Architecture

### Triple couche de protection

| Couche | Composant | Mécanisme |
|--------|-----------|-----------|
| **API** | `onRequest` hook global (app.ts) | HTTP 402 si tenant locked |
| **Client layout** | `useEntitlement` hook (ClientLayout.tsx) | Redirect vers `/locked` |
| **Sidebar** | Nav items disabled + icône Lock | UX cohérente |

### Flux décisionnel `getTenantEntitlement(tenantId)`

```
billing_subscriptions.status ?
├── active → isLocked=false
├── trialing → trial expiré ? → grace expiré ? → LOCKED (TRIAL_EXPIRED) : ok
├── past_due → grace 3j expiré ? → LOCKED (PAST_DUE) : ok (grace)
├── canceled → LOCKED (CANCELED)
├── incomplete/unpaid → LOCKED (PAST_DUE)
└── pas de subscription → legacy tenant ? → ok si active : LOCKED
```

### Scénarios de lock

| Scénario | isLocked | lockReason | HTTP Status |
|----------|----------|------------|-------------|
| **Canceled** | `true` | `CANCELED` | **402** |
| **Past due (in grace, J-3)** | `false` | `NONE` | 404 (passthrough) |
| **Past due (after grace, J+2)** | `true` | `PAST_DUE` | **402** |
| **Trialing (actif)** | `false` | `NONE` | passthrough |
| **Active** | `false` | `NONE` | passthrough |

### Réponse 402

```json
{
  "error": "PAYMENT_REQUIRED",
  "lockReason": "CANCELED",
  "billingStatus": "canceled",
  "billingUrl": "/billing/plan",
  "message": "Accès bloqué. Veuillez vérifier votre abonnement."
}
```

## Client UI

### Bandeau trial (bleu)
- Visible quand `billingStatus === 'trialing'` et `daysLeftTrial > 0`
- Texte : "Essai gratuit actif — J-14 restants"
- CTA : "Finaliser l'abonnement" → `/billing/plan`

### Bandeau grace (orange/ambre)
- Visible quand `billingStatus === 'past_due'` et dans la période de grâce
- Texte : "Paiement à régulariser — J-2 avant blocage"
- CTA : "Mettre à jour le paiement" → `/billing/plan`

### Page /locked
- Messages contextuels selon `lockReason` (TRIAL_EXPIRED, PAST_DUE, CANCELED, NO_SUBSCRIPTION)
- CTA principal : "Souscrire un abonnement" ou "Mettre à jour mon paiement"
- Bouton "Vérifier à nouveau" (re-fetch entitlement)
- Bouton "Se déconnecter"
- Info : "Vos données sont conservées"

### Sidebar
- Nav items disabled (grisés + icône Lock) quand tenant locked
- Sauf : Facturation, Aide, Déconnexion

## Grace period

- **Durée** : 3 jours
- **Référence** : `billing_subscriptions.updated_at` (approximation de quand le statut est devenu past_due)
- **Trial end** : calculé comme `tenants.created_at + 14 jours` (fallback si `current_period_end` est null)
