## 6. CAUSE RACINE UNIQUE

**Le tenant `compta-ecomlg-gmail--mnkjttw7` n'existait pas dans la table `Tenant` de la DB backend (Prisma).**

### Explication

Le système utilise **deux bases de données** :

1. **Product DB** (`keybuzz` PostgreSQL) : contient `tenant_channels`, `tenants`, `users`, etc. Les tenant IDs sont des slugs comme `compta-ecomlg-gmail--mnkjttw7`.
2. **Backend DB** (Prisma) : contient `MarketplaceConnection`, `InboundConnection`, `InboundAddress`, `OAuthState`, `Tenant`. Les tenant IDs sont des slugs OU des CUIDs.

Le modèle `InboundConnection` a une **FK contrainte** vers `Tenant.id` dans la DB Prisma :

```
tenant  Tenant  @relation(fields: [tenantId], references: [id], onDelete: Cascade)
```

Mais le modèle `MarketplaceConnection` n'a **PAS de FK** — c'est un simple champ `String tenantId`.

Quand un nouveau tenant est créé via la product DB (API), il **n'est PAS automatiquement répliqué** dans la table `Tenant` de la DB backend Prisma. Résultat :

- OAuth callback → `completeAmazonOAuth` → crée/met à jour `MarketplaceConnection` → **OK** (pas de FK)
- OAuth callback → `ensureInboundConnection` → tente de créer `InboundConnection` → **FK VIOLATION** (tenant absent de Prisma)

### Tenants backend Prisma vs Product DB

La DB Prisma ne contenait que 15 tenants (créés manuellement ou par des scripts anciens). Le tenant `compta-ecomlg-gmail--mnkjttw7` n'y figurait pas.

## 7. FIX APPLIQUÉ

### Fichier modifié

`keybuzz-backend/src/modules/marketplaces/amazon/amazon.routes.ts` (ligne 340)

### Code ajouté (PH145.6)

```typescript
// PH145.6: Ensure tenant exists in backend Prisma DB (FK for InboundConnection)
await prisma.tenant.upsert({
  where: { id: oauthState.tenantId },
  create: {
    id: oauthState.tenantId,
    slug: oauthState.tenantId,
    name: oauthState.tenantId,
  },
  update: {},
});
console.log(`[Amazon OAuth] PH145.6: Tenant ensured in backend DB: ${oauthState.tenantId}`);
```

### Position dans le flow

```

  ↓
Get countries from tenant_channels
  ↓
★ prisma.tenant.upsert()       ← PH145.6: Ensure tenant exists
  ↓
ensureInboundConnection()       ← Plus de FK violation
  ↓
prisma.inboundConnection.update (READY)
  ↓
prisma.inboundAddress.findMany
  ↓
productDb UPDATE tenant_channels (atomic per country_code)
```
