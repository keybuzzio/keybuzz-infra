## 4. PROBLÈMES CRITIQUES IDENTIFIÉS

### ❌ Problème 1 : Incohérence SLA
- Dashboard : `LOWER(sla_state) = 'risk'`
- KPI : `sla_state = 'AT_RISK'`
- Si la DB contient `'AT_RISK'`, le Dashboard ne matchera pas avec `LOWER('AT_RISK') = 'at_risk'`

### ❌ Problème 2 : Endpoints multiples
- 3 endpoints différents pour les stats SLA
- 2 endpoints différents pour les counts conversations
- Risque de divergence si une requête est modifiée

### ❌ Problème 3 : Page Orders manquante
- Pas de composant Orders dans le frontend
- Les métriques Orders du Dashboard ne sont pas exposées ailleurs
- Le backfill ajoute des orders mais aucune UI pour les voir

### ❌ Problème 4 : Calculs frontend
- Calculs locaux en mode fallback
- Possibilité de mismatch entre frontend et backend
