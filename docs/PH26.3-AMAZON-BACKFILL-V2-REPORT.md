## 1. Etat Initial

### Architecture
L'architecture etait deja separee en 2 workers distincts (PH26.3 initial) :
- **Orders Worker** : 2 replicas, boucle infinie, traitement par chunks de 30 jours
- **Items Worker** : 1 replica, remplissage asynchrone des items manquants

### Bugs critiques decouverts

| # | Bug | Impact | Severite |
|---|-----|--------|----------|
| 1 | `unitPrice: Infinity` dans Items Worker | 20 orders echouent par batch, lock bloque | **CRITIQUE** |
| 2 | SQL injection dans `updateBackfillState()` | `$executeRawUnsafe` avec concatenation de strings | **MAJEUR** |
| 3 | Enum cast manquant (BackfillStatus) | Worker crash sur chaque update d'etat | **CRITIQUE** |
| 4 | Backfill bloque en ERROR | ecomlg-001 a 33% depuis le 30 jan 2026 | **MAJEUR** |
| 5 | 590+ commandes sans produits | Items worker ne peut pas les traiter | **MOYEN** |

### Cause du bug Infinity

```typescript
// AVANT (BUG) — division par 0 quand QuantityOrdered = 0
unitPrice: item.ItemPrice ? parseFloat(item.ItemPrice.Amount) / item.QuantityOrdered : 0
```

Amazon retourne parfois `QuantityOrdered: 0` pour des commandes annulees. La division produit `Infinity`, Prisma rejette l'insertion, le batch entier echoue.

### Cause du bug SQL injection

```typescript
// AVANT (BUG) — concatenation directe de valeurs utilisateur
setParts.push(`"nextToken" = '${updates.nextToken.replace(/'/g, "''")}'`);
setParts.push(`"lastError" = '${updates.lastError.replace(/'/g, "''").substring(0, 500)}'`);
await prisma.$executeRawUnsafe(`UPDATE ... SET ${setParts.join(', ')} WHERE ...`);
```

Le `nextToken` et `lastError` proviennent de reponses Amazon non-sanitisees.

## 3. Corrections Appliquees

### Fix 1 : Safe Unit Price (Infinity guard)

```typescript
// APRES (SAFE) — guards contre Infinity, NaN, negatif
function safeUnitPrice(itemPrice: any, quantity: number): number {
  if (!itemPrice?.Amount) return 0;
  const qty = quantity > 0 ? quantity : 1;
  const raw = parseFloat(itemPrice.Amount);
  if (!Number.isFinite(raw)) return 0;
  const price = raw / qty;
  if (!Number.isFinite(price) || price < 0) return 0;
  return Math.round(price * 100) / 100;
}
```

### Fix 2 : SQL Injection → Parameterized Query

```typescript
// APRES (SAFE) — placeholders $1..$N avec cast enum explicite
await prisma.$executeRawUnsafe(
  `UPDATE "amazon_orders_backfill_state"
   SET "status" = $1::"BackfillStatus",
       "progressPercent" = $2, ...
   WHERE "tenantId" = $14 AND "marketplace" = 'AMAZON'`,
  newStatus, newProgress, ..., tenantId
);
```

### Fix 3 : Lock Cleanup Automatique

```typescript
// Nettoyage des locks expires a chaque iteration
async function cleanupExpiredLocks(): Promise<void> {
  const cleaned = await prisma.$executeRaw`
    DELETE FROM "amazon_backfill_locks" WHERE "expiresAt" < NOW()
  `;
  if (cleaned > 0) console.log(`[BackfillLock] CLEANUP expired=${cleaned}`);
}
```

## 5. Limites Amazon API

| Limite | Valeur | Mitigation |
|--------|--------|-----------|
| Orders API rate | ~1 req/600ms | `RATE_LIMIT_DELAY_MS=600` |
| Items API rate | ~30 req/min | `MAX_CALLS_PER_MINUTE=30` |
| 429 backoff | exponentiel | 8s × attempt (orders), 5s × attempt (items) |
| Max retries | 5 (orders), 3 (items) | Puis ERROR avec retryAfter |
| Batch size | 100 orders/page | SP-API MaxResultsPerPage |
