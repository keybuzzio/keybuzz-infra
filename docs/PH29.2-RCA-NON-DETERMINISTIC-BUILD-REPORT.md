## Investigation

### A) Vérification filesystem source

```bash
$ ls -la /root/keybuzz-client/public/
# Résultat AVANT fix: branding/ et marketplaces/ ABSENTS
```

### B) Vérification Git

```bash
$ git ls-files public/branding public/marketplaces
# Résultat AVANT fix: AUCUN FICHIER TROUVÉ
```

**Constat : Ces fichiers n'étaient PAS versionnés dans Git.**

### C) Vérification Dockerfile

```dockerfile
COPY . .                                    # Copie source
COPY --from=builder /app/public ./public    # Copie public
```

**Le Dockerfile est correct.** Le problème vient du source, pas du build.

### D) Origine des assets GOLDEN

Les fichiers ont été ajoutés manuellement au build context à des dates différentes :
- `branding/` : Jan 20 22:13
- `marketplaces/` : Jan 22 03:54

Ces fichiers existaient localement lors du build GOLDEN mais n'avaient jamais été committés.

---

## CAUSE RACINE (UNIQUE)

**Les assets `branding/` et `marketplaces/` n'étaient PAS versionnés dans Git.**

Le build GOLDEN avait été effectué depuis un environnement où ces fichiers existaient localement (ajoutés manuellement). Chaque nouveau build depuis un clone Git propre ne contient pas ces fichiers.

---

## Fix Appliqué

```bash
# Extraction des assets depuis GOLDEN
docker cp $(docker create ghcr.io/keybuzzio/keybuzz-client:menu-fr-ph28.26-2026-02-03):/app/public/branding ./public/
docker cp $(docker create ghcr.io/keybuzzio/keybuzz-client:menu-fr-ph28.26-2026-02-03):/app/public/marketplaces ./public/

# Commit dans Git
git add public/branding public/marketplaces
git commit -m "fix: add missing branding and marketplace assets to Git"
git push origin main
```
