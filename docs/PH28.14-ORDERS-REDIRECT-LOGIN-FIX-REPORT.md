# PH28.14 - Orders Redirect to Login Fix Report

**Date:** 2026-02-03
**Status:** ✅ COMPLETED
**Environments:** DEV + PROD

---

## Objectif

Après signup (Google ou OTP), cliquer sur "Commandes" ne doit plus rediriger vers `/login`.

**Comportement attendu :**
- Si user non authentifié → `/login` (OK)
- Si user authentifié + tenant → page commandes avec données ou empty state
- Si API error → message d'erreur (pas de redirect)

---

## Cause Racine

### Fichiers affectés

| Fichier | Ligne | Code problématique |
|---------|-------|-------------------|
| `app/orders/page.tsx` | 265 | `window.location.href = '/login';` |
| `app/orders/[orderId]/page.tsx` | 71 | `router.push('/login');` |

### Problème

Les deux pages "Commandes" redirigeaient vers `/login` dès qu'une réponse API retournait un status 401, même si l'utilisateur était authentifié.

```typescript
// AVANT (problématique)
if (response.status === 401) {
  window.location.href = '/login';  // Redirection incorrecte
  return;
}
```

### Scénario du bug

1. User signup OTP → tenant créé
2. Arrive sur dashboard (authentifié)
3. Clique sur "Commandes"
4. API `/api/orders` retourne 401 (session transitoire ou erreur API)
5. Page redirige vers `/login` au lieu d'afficher une erreur

---

## Fix Appliqué

### Modification

Remplacement de la redirection par un `throw Error` qui sera géré par le catch block et affichera un message d'erreur à l'utilisateur.

```typescript
// APRÈS (corrigé)
if (response.status === 401) {
  // PH28.14: Show error instead of redirect to /login
  throw new Error('Session ou API indisponible');
}
```

### Fichiers modifiés

- `/opt/keybuzz/keybuzz-client/app/orders/page.tsx` (ligne 265-266)
- `/opt/keybuzz/keybuzz-client/app/orders/[orderId]/page.tsx` (ligne 71-72)
