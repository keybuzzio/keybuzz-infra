## 1. Résumé exécutif

Le bug de classification des brouillons Autopilot est corrigé. Avant ce fix, des brouillons contenant des **promesses d'action humaine** ("je vais vérifier", "je reviens vers vous", "je vais m'assurer") étaient classés comme `DRAFT_GENERATED` au lieu de `ESCALATION_DRAFT`, ce qui empêchait toute escalade lors de leur validation.

---

## 2. Reproduction exacte

### Avant le fix

Un brouillon généré pour une conversation de type "réparation produit" :
- Contenu : "Je vais m'assurer que votre dossier est traité..."
- `blocked_reason` : **`DRAFT_GENERATED`** ❌
- `needsHumanAction` : `true` (stocké dans metadata)
- `falsePromisePatterns` : `["je reviens vers vous"]`
- Escalade au consume : **AUCUNE** ❌

Preuve en DB (avant fix) :
```json
{
  "id": "alog-1775481882073-g2r2arbfq",
  "blocked_reason": "DRAFT_GENERATED",
  "needsHumanAction": true,
  "patterns": ["je reviens vers vous"]
}
```

---

## 3. Cause racine

**Fichier** : `keybuzz-api/src/modules/autopilot/engine.ts` — bloc safe_mode (lignes 345-362)

La fonction `detectFalsePromises()` était correctement exécutée et le flag `needsHumanAction` stocké dans les métadonnées du log. Cependant, le `blocked_reason` restait **toujours** `DRAFT_GENERATED`, sans aucune logique de reclassification.

```typescript
// AVANT (bugué) :
const falsePromises = detectFalsePromises(suggestion.replyContent || '');
const needsHumanAction = falsePromises.length > 0;
// needsHumanAction est true MAIS :
await logAction(..., 'DRAFT_GENERATED', ...);  // ← toujours DRAFT_GENERATED
return { reason: 'DRAFT_GENERATED', escalated: false };
```

---

## 4. Règle métier appliquée

### Si le brouillon contient une promesse d'action humaine
- Classé comme `ESCALATION_DRAFT:confidence`
- `escalationTarget` : configuré dans autopilot_settings
- `escalationReason` : `Promesse d'action détectée: {patterns}`
- `action` : `escalate`
- Au consume `applied` → escalade effective (fix E.6)

### Si le brouillon est une simple réponse informative
- Reste `DRAFT_GENERATED`
- Pas d'escalade au consume

### Patterns de promesses détectés (15 patterns)
1. `je vais (re)contacter`
2. `je vais vérifier`
3. `je me renseigne`
4. `je reviens vers vous`
5. `je vais m'assurer`
6. `je prends contact`
7. `je vais investiguer`
8. `je vais faire le nécessaire`
9. `je vais transmettre`
10. `je vais immédiatement vérifier`
11. `je vais traiter`
12. `nous allons procéder/vérifier/investiguer/contacter`
13. `je vais consulter`
14. `je vais escalader`
15. `votre dossier est/sera en cours/pris en charge/traité`
