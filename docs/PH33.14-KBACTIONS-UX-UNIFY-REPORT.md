## Objectif

Supprimer la confusion entre **dotation mensuelle incluse** (selon le plan) et **solde actuel** (crédit packs) en unifiant le wording sur les 3 pages billing.

---

## Modifications par page

### 1. Facturation principale (`app/billing/page.tsx`)

| Avant | Après |
|-------|-------|
| `Dotation : X KBActions / mois` | `Dotation mensuelle incluse : X KBActions/mois` |
| `Solde : Y` | `Solde actuel : Y` |
| *(pas d'explication)* | `La dotation est incluse dans votre abonnement. Le solde correspond à votre crédit restant (packs).` |

### 2. Mon plan (`app/billing/plan/page.tsx`)

| Avant | Après |
|-------|-------|
| `Dotation mensuelle : X KBActions/mois` | `Dotation mensuelle incluse : X KBActions/mois` |
| *(pas d'explication)* | `La dotation est incluse dans votre abonnement. Le solde correspond à votre crédit restant (packs).` |

### 3. KBActions détaillé (`app/billing/ai/page.tsx`)

| Avant | Après |
|-------|-------|
| `Solde KBActions` (titre) | `Solde actuel` |
| `KBActions restantes` (sous-titre) | `KBActions restantes (crédit packs)` |
| `Incluses/mois: {backend value}` | `Dotation/mois : {planCapabilities value}` (source unique) |
| `Achetees: X` | `Packs achetés : X` |
| `Votre solde et consommation` | `Solde actuel et consommation` |
| *(pas de bloc dotation)* | Bloc vert `Dotation mensuelle incluse : X KBActions/mois` |
| *(pas d'explication)* | `La dotation est incluse dans votre abonnement. Le solde correspond à votre crédit restant (packs).` |

**Source unique de la dotation** : `PLAN_CAPABILITIES[plan].kbActionsMonthly` depuis `planCapabilities.ts` — aucun hardcode.
