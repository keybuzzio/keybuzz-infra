## 1. Objectif

Ajouter un mecanisme de controle pilotable par tenant permettant de separer :
- **Collecte** des signaux d'apprentissage (PH51) : toujours active par defaut
- **Application** des signaux dans la reponse IA (PH52) : desactivee par defaut, activable par toggle

## 2. Modele de Donnees

### Table : `tenant_ai_learning_settings`

| Colonne | Type | Default | Description |
|---|---|---|---|
| `tenant_id` | TEXT PK | - | ID du tenant |
| `learning_collect_enabled` | BOOLEAN | `true` | PH51 collecte les signaux |
| `learning_apply_enabled` | BOOLEAN | `false` | PH52 applique les signaux |
| `created_at` | TIMESTAMPTZ | `NOW()` | Date de creation |
| `updated_at` | TIMESTAMPTZ | `NOW()` | Derniere mise a jour |

### Comportement par defaut (sans ligne en DB)
- `collectEnabled = true` : PH51 collecte toujours
- `applyEnabled = false` : PH52 n'influence pas l'IA

## 3. Service

**Fichier** : `src/services/learningControlService.ts`

| Fonction | Description |
|---|---|
| `getLearningControl(tenantId)` | Retourne l'etat du toggle (DB, cache ou default) |
| `isLearningCollectEnabled(tenantId)` | Shortcut pour verifier la collecte |
| `isLearningApplyEnabled(tenantId)` | Shortcut pour verifier l'application |
| `updateLearningControl(tenantId, collect, apply)` | Met a jour le toggle (upsert + invalidate cache) |

### Cache Redis
- Cle : `learning_control:{tenantId}`
- TTL : 5 minutes
- Invalidation : automatique apres `updateLearningControl()`

## 4. Comportement ON/OFF

### Etat : `applyEnabled = false` (defaut)

```
PH51 (Learning Loop)  →  collecte active     ✓
PH52 (Adaptive)       →  prompt non injecte  ✗
decisionContext        →  adaptiveResponse.enabled = false
                          adaptiveResponse.reason = "disabled_by_tenant_toggle"
                          learningControl.collectEnabled = true
                          learningControl.applyEnabled = false
```

### Etat : `applyEnabled = true`

```
PH51 (Learning Loop)  →  collecte active     ✓
PH52 (Adaptive)       →  prompt injecte      ✓
decisionContext        →  adaptiveResponse.enabled = true
                          learningControl.collectEnabled = true
                          learningControl.applyEnabled = true
```

## 5. Integration Pipeline IA

Le toggle est verifie dans `ai-assist-routes.ts` :

```
Pipeline:
1  Base Prompt
2  SAV Policy (PH41)
3  Tenant Policy (PH44)
4  Historical Engine (PH43)
5  Decision Tree (PH45)
6  Response Strategy (PH46)
7  Refund Protection (PH49)
8  Merchant Behavior (PH50)
9  [TOGGLE CHECK] → si applyEnabled: Adaptive Response (PH52)
10 Order Context
11 Supplier Context
12 Tenant Rules
```

PH51 continue de collecter independamment du toggle.
