## Cause racine

**Race condition dans `select-tenant/page.tsx`** :

1. L'utilisateur arrive sur `/select-tenant` après login
2. `authLoading` passe à `false` (session Next-Auth prête côté client)
3. Le fetch vers `/api/auth/tenants` est lancé immédiatement
4. **Mais** la session n'est pas encore propagée au backend → le backend répond 401 ou timeout
5. Le code **ne faisait qu'un seul essai** et affichait immédiatement `"Impossible de charger vos espaces"` en rouge
6. L'utilisateur voyait une bannière d'erreur agressive alors que l'API fonctionnait (il suffisait de retry)

**Problème secondaire dans `TenantProvider.tsx`** :
Le message d'erreur après retry exposait le message technique brut (`Failed to fetch tenant context`) au lieu d'un message utilisateur.

---

## Fix appliqué

### 1. `app/select-tenant/page.tsx` — Refonte complète du fetch

| Aspect | Avant | Après |
|--------|-------|-------|
| **Tentatives** | 1 seule | 4 (1 + 3 retries) |
| **Délai entre retries** | Aucun | 1.5s (`RETRY_DELAY_MS`) |
| **Message d'erreur** | `"Impossible de charger vos espaces"` (rouge) | `"Connexion temporairement instable"` (ambre) |
| **Couleur bannière** | `bg-red-500` | `bg-amber-500` (non agressif) |
| **Bouton retry** | Aucun | Bouton "Réessayer" avec icône RefreshCw |
| **Message loading** | `"Chargement..."` | `"Chargement de vos espaces…"` |
| **Empty state** | Texte seul | Icône Building2 + texte propre |
| **Moment de l'erreur** | Dès le premier échec | Seulement après épuisement des 4 tentatives |

Pattern implémenté :
```
Loading → fetch (attempt 1) → fail → wait 1.5s → fetch (attempt 2) → fail → wait 1.5s → ... → soft error
```

L'état `isLoadingSpaces` reste `true` pendant TOUT le cycle de retry. Le spinner reste affiché, aucune erreur n'apparaît prématurément.

### 2. `src/features/tenant/TenantProvider.tsx` — Message d'erreur unifié

| Avant | Après |
|-------|-------|
| `lastError.message \|\| 'Failed to fetch tenant context'` | `'Connexion temporairement instable'` |

## Comportement attendu après fix

| Scénario | Résultat |
|----------|----------|
| Login normal, API répond 200 au 1er essai | Spinner → liste des espaces (pas d'erreur) |
| Login, API lente (200 après 2s) | Spinner → liste (retry transparent) |
| Login, API 401 au 1er essai puis 200 | Spinner → liste (retry transparent) |
| API totalement down | Spinner (~6s) → bannière ambre "Connexion temporairement instable" + bouton Réessayer |
| 0 tenants, API 200 | Icône + "Aucun espace trouvé" + bouton "Créer un espace" |
