## Décisions techniques appliquées

### 1️⃣ Orders = Chemin critique (FAST PATH)

| Aspect | Implémentation |
|--------|----------------|
| **Traitement** | Orders-only, bulk upsert 300/batch |
| **AUCUN appel OrderItems** | ✅ Confirmé |
| **Idempotence** | `tenantId + marketplace + externalOrderId` |
| **Vitesse mesurée** | **1,187 orders/min** |

### 2️⃣ Items = Background ASYNC

| Aspect | Implémentation |
|--------|----------------|
| **Worker séparé** | `amazon-items-worker` (1 replica) |
| **Rate-limit strict** | 30 calls/min/tenant |
| **Retry contrôlé** | Max 3 retries, backoff exponentiel |
| **Non-bloquant** | Items peuvent prendre des heures |

### 3️⃣ Multi-tenant strict

| Aspect | Implémentation |
|--------|----------------|
| **Isolation logique** | Table `amazon_backfill_locks` |
| **1 tenant = 1 worker** | `FOR UPDATE SKIP LOCKED` + locks |
| **Quotas par tenant** | Configurable via table |
| **Tenant saturé** | Ne bloque JAMAIS les autres |

### 4️⃣ Observabilité FIABLE

| Métrique | Endpoint |
|----------|----------|
| `orders_total` | `/api/v1/orders/backfill/metrics` |
| `orders_persisted` | `/api/v1/orders/backfill/metrics` |
| `items_total_expected` | `/api/v1/orders/backfill/metrics` |
| `items_pending` | `/api/v1/orders/backfill/metrics` |
| `items_failed` | `/api/v1/orders/backfill/metrics` |
| `last_backfill_at` | `/api/v1/orders/backfill/metrics` |

## Limites connues

1. **Rate limiting SP-API** : ~100 req/min max, goulot d'étranglement principal
2. **Items fill lent** : 30 calls/min/tenant = ~20 orders/min pour items
3. **Retries automatiques** : Nécessitent intervention manuelle après 5 erreurs consécutives (retryAfter)

## Schéma du flow

```
1. POST /backfill/start?tenantId=X
   └─> Retourne 202, état = PENDING

2. Orders Worker (K8s pod)
   └─> SELECT ... FOR UPDATE SKIP LOCKED
   └─> Acquire lock (amazon_backfill_locks)
   └─> Fetch orders SP-API (chunk 30 jours)
   └─> Bulk upsert 300 orders/batch
   └─> Update state (progress, nextChunk)
   └─> Release lock
   └─> Repeat

3. Items Worker (K8s pod)
   └─> SELECT orders without items LIMIT 20
   └─> Acquire lock (ITEMS)
   └─> Fetch items SP-API (rate limited)
   └─> Insert items
   └─> Release lock
   └─> Repeat

4. GET /backfill/metrics
   └─> Retourne métriques FIABLES
```
