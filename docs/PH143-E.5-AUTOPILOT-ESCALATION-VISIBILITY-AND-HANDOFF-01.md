## Problemes Identifies et Corriges

### 1. `escalation_target` absent des queries SQL (API)

**Cause** : Les SELECT des routes `/messages/conversations` (liste et detail) incluaient `escalation_status, escalation_reason, escalated_at, escalated_by_type` mais **pas** `escalation_target`.

**Fix** (`src/modules/messages/routes.ts`) :
- Ajout de `escalation_target` dans les 3 queries SELECT (liste, detail, GET escalation)
- Ajout de `escalationTarget` dans la reponse JSON du GET `/escalation`

### 2. `escalationTarget` absent du mapping client

**Cause** : Le service `conversations.service.ts` et `InboxTripane.tsx` ne mappaient pas la colonne `escalation_target` vers la propriete `escalationTarget`.

**Fix** :
- `src/services/conversations.service.ts` : ajout `escalationTarget` dans le type `Conversation` et le mapping
- `app/inbox/InboxTripane.tsx` : ajout `escalationTarget` dans `LocalConversation`, le mapping liste, et les props du `EscalationPanel`

### 3. Cible d'escalade non affichee dans l'UI

**Cause** : Le `EscalationPanel` affichait le statut et la raison mais pas la cible.

**Fix** (`src/features/inbox/components/EscalationPanel.tsx`) :
- Ajout du dictionnaire `ESCALATION_TARGET_LABELS` (client → "Votre equipe", keybuzz → "KeyBuzz", both → "Les deux")
- Ajout de la prop `escalationTarget`
- Affichage d'un badge bleu avec la cible quand une escalade est active

## Visibilite UI de l'Escalade

### Badge dans la liste (inbox)
- `EscalationBadge` : affiche "Escalade" (rouge) ou "Recommande" (ambre)
- Visible dans la liste de conversations pour chaque conversation escaladee

### Panel dans le detail
- `EscalationPanel` : statut, raison, **cible** (nouveau), boutons escalade/de-escalade
- Badge cible en bleu : "Votre equipe" / "KeyBuzz" / "Les deux"

### Assignation
- `AssignmentPanel` : badge "IA" (indigo) ou "Assigne" (vert)
- Bouton "Prendre la main" : disponible si conversation non assignee a l'utilisateur
- Bouton "Relacher vers l'IA" : disponible si conversation assignee a l'utilisateur

### Tri
- Conversations escaladees sont triees en haut de la liste (score 0)
- Conversations recommandees juste apres (score 1)
- Filtre "A reprendre" disponible pour voir les escaladees non assignees

## Workflow Complet Valide

### Mode Safe (safe_mode: true)

```
Message client → autopilot evaluate
  → IA genere reponse (confiance >= 0.5)
  → DRAFT_GENERATED (status=skipped)
  → Drawer auto-open avec draft
  → Agent valide → message envoye
  
  → IA confiance < 0.5 (ou action 'none')
  → ESCALATION_DRAFT avec fallback reply
  → Drawer auto-open avec draft de temporisation
  → Agent valide → message envoye + conversation escaladee
```

### Mode Autonome (allow_auto_reply + allow_auto_escalate)

```
Message client → autopilot evaluate
  → IA genere reponse (confiance >= 0.5)
  → Message envoye automatiquement
  
  → IA confiance < 0.5
  → ESCALATION_WITH_REPLY
  → Message de temporisation envoye + conversation escaladee
  → Agent voit la conversation dans "A reprendre"
```

### Handoff Agent

```
Conversation escaladee → visible dans inbox avec badge rouge
  → Agent clique "Prendre la main" (AssignmentPanel)
  → assigned_agent_id = userId
  → assignedType = 'human'
  → Agent repond normalement
  → Conversation peut etre de-escaladee ou resolue
```
