**Date**: 2026-02-02

## Résumé Exécutif

Remplacement du "mode dev" OTP par un système de production avec envoi email réel via SMTP, rate limiting, hash storage, et whitelist PROD pour la sécurité.

---

## 1. Changements Effectués

### 1.1 Service OTP Production-Ready

**Fichier**: `keybuzz-client/src/lib/otp-store.ts`

| Aspect | Avant (DEV) | Après (PROD) |
|--------|-------------|--------------|
| Génération OTP | `Math.random()` | `crypto.randomBytes()` |
| Stockage | Texte clair | Hash SHA-256 + salt |
| Expiration | 10 min | 10 min |
| Max tentatives | 3 | 5 |
| Rate limit email | Basique | 3 OTP / 15 min / email |
| Rate limit IP | Aucun | 10 OTP / heure / IP |

### 1.2 Envoi Email OTP via SMTP

**Fichier**: `keybuzz-client/app/api/auth/magic/start/route.ts`

- Intégration `nodemailer` avec import dynamique
- SMTP Provider: `mail.keybuzz.io:25`
- From: `no-reply@keybuzz.io`
- Template HTML/texte professionnel en français

### 1.3 Sécurité PROD

- **Logs PROD**: Email masqué (`lud***@gmail.com`), code JAMAIS loggé
- **Logs DEV**: Code affiché pour faciliter tests
- **Whitelist PROD**: Emails autorisés configurés via `OTP_EMAIL_WHITELIST`

### 1.4 Modifications Dockerfile

```dockerfile

COPY --from=builder /app/node_modules/nodemailer ./node_modules/nodemailer
```

## 8. Rollback Procedure

### Rollback DEV

```bash
# Revert to previous image
kubectl set image deployment/keybuzz-client \
  keybuzz-client=ghcr.io/keybuzzio/keybuzz-client:dev-tenant-gate-2026-02-02a \
  -n keybuzz-client-dev

# Or via ArgoCD (reverts automatically from git)
cd /opt/keybuzz/keybuzz-infra
git revert HEAD
git push origin main
```

### Rollback PROD

```bash
kubectl set image deployment/keybuzz-client \
  keybuzz-client=ghcr.io/keybuzzio/keybuzz-client:prod-tenant-gate-2026-02-02a \
  -n keybuzz-client-prod

# Remove SMTP env vars
kubectl set env deployment/keybuzz-client -n keybuzz-client-prod \
  SMTP_HOST- SMTP_PORT- OTP_FROM_EMAIL- OTP_EMAIL_WHITELIST-
```

## 9. Actions Post-Déploiement

### Whitelist Temporaire

La whitelist `OTP_EMAIL_WHITELIST` est temporaire et doit être supprimée une fois les tests terminés :

```bash
# Pour désactiver la whitelist (autoriser tous les emails)
kubectl set env deployment/keybuzz-client -n keybuzz-client-prod OTP_EMAIL_WHITELIST-
```

### Pattern Regex Note

Le pattern `ludovic+otp-test*@ecomlg.fr` dans la whitelist ne fonctionne pas correctement car le `+` doit être échappé en regex. Pour les tests PROD, utiliser les emails exactement whitelistés :
- `ludovic@ecomlg.fr`
- `ludo.gonthier@gmail.com`
