## 1. DÉFINITION "PROD GOLDEN MINIMAL"

Le minimum viable pour qu'un utilisateur puisse:
1. S'inscrire
2. Se connecter
3. Créer/rejoindre un tenant
4. Voir son dashboard (vide ou non)
5. Voir son inbox (vide ou non)

## 6. TAG STRATEGY

### 6.1 Nomenclature

```
prod-YYYY-MM-DD[-suffix]
```

Exemples:
- `prod-2026-02-01` - Premier tag PROD
- `prod-2026-02-01-hotfix1` - Hotfix si nécessaire

### 6.2 Source du Tag

```
DEV stable commit → Tag prod-YYYY-MM-DD → Deploy PROD
```

**JAMAIS:**
- Branche en cours
- Commit non testé
- Code non mergé
