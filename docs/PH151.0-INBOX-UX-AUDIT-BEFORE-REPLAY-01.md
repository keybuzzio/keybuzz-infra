## 6. Risques de regression

### CRITIQUE


| Risque                      | Cause historique                                                | Mitigation                                                              |
| --------------------------- | --------------------------------------------------------------- | ----------------------------------------------------------------------- |
| **Ecran blanc React**       | PH151.3 — MessageBubble crash sur messages avec properties null | ErrorBoundary autour du rendu message + guard `msg?.content`            |
| **Scope contamination**     | PH151.3 — fichiers non commites inclus dans le build            | GitOps strict : commit AVANT build, git diff --stat verification        |
| **InboxTripane regression** | 1964 lignes, fichier monolithique                               | Modifications incrementales, commit par feature, test apres chaque step |


### ELEVE


| Risque                               | Details                                                                                   | Mitigation                                                                                |
| ------------------------------------ | ----------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| MessageBubble remplace rendu inline  | Le rendu inline actuel (lignes 1637-1750) fonctionne - le remplacer en bloc est dangereux | Integration progressive : garder l'inline en fallback, tester sur plusieurs conversations |
| Conversations avec donnees atypiques | Messages `direction` null, `content` null, `visibility` undefined                         | Guards defensifs dans messageClassifier et MessageBubble                                  |
| AICaseSummary charge avant messages  | Si `messages` array vide, crash potentiel                                                 | Guard `if (!messages?.length) return null`                                                |


### MOYEN


| Risque                                  | Details                                                                          |
| --------------------------------------- | -------------------------------------------------------------------------------- |
| OrderSidePanel collapsible              | Modifier un composant de 803 lignes - tester collapse/expand dans tous les etats |
| ConversationSummaryBar props manquantes | Certaines conversations n'ont pas `subject`, `savStatus`, `escalationStatus`     |

## 7. Plan de replay recommande

### Approche : incrementale, 4 etapes distinctes avec build + validation entre chaque

#### STEP 1 — Classification + MessageBubble (risque le plus eleve en premier)

- Creer `messageClassifier.ts`
- Creer `MessageBubble.tsx` avec ErrorBoundary interne
- Integrer dans InboxTripane (remplacer le rendu inline)
- **VALIDATION OBLIGATOIRE** : ouvrir 5+ conversations differentes, verifier console 0 erreurs

#### STEP 2 — Contexte (SummaryBar + CaseSummary + FilterToggle)

- Creer `ConversationSummaryBar.tsx` (version PH151.1 simplifiee)
- Creer `AICaseSummary.tsx` (version PH151.2 data-driven)
- Creer `MessageFilterToggle.tsx`
- Creer state `hideAmazonAuto`
- Integrer dans InboxTripane
- **VALIDATION** : verifier barre contexte + resume IA + toggle filtre

#### STEP 3 — Sidebar Intelligence

- Modifier `OrderSidePanel.tsx` : ajouter topSlot + collapsible
- Passer AICaseSummary dans topSlot depuis InboxTripane
- **VALIDATION** : collapse/expand sidebar, resume IA visible dans tous les etats

#### STEP 4 — Priority UX (PH151.3)

- Ajouter `PriorityDot` dans `PriorityBadge.tsx`
- Remplacer `PriorityBadge` par `PriorityDot` dans InboxTripane
- Simplifier bloc stats priority
- **VALIDATION** : dots couleur visibles, stats compactes

### Alternative : scope reduit

Si le replay complet est trop risque, commencer par STEP 1 seul (MessageBubble + classification) et valider avant de continuer. Le reste peut etre fait en phases separees.
