## Analyse racine

### Chronologie des evenements

1. **PH151.2** — Build `v3.5.63-ph151.2-case-summary-clean-dev` avec case summary fonctionnel
2. **PH151.3** — Build `v3.5.64-ph151.3-priority-clean-dev` — REGRESSE (contamination scope PH151)
3. **PH151.3.1** — Rollback DEV vers `v3.5.63-ph151.2-case-summary-clean-dev` — CORRECT
4. **PH151.3A** — Build `v3.5.64-ph151.3a-priority-dot-dev` — **a partir d'un `git checkout -- InboxTripane.tsx`** sur le bastion, qui a PERDU les modifications case summary non commitees

### Cause racine

Les modifications `InboxTripane.tsx` de PH151.2 (case summary data-driven) n'etaient **jamais commitees dans Git** sur le bastion. Elles existaient uniquement comme modifications locales non trackees.

Quand PH151.3A a execute `git checkout -- app/inbox/InboxTripane.tsx` pour nettoyer la contamination PH151.3, cette commande a retabli le fichier dans l'etat Git HEAD — un etat qui ne contenait PAS les modifications case summary de PH151.2.

Le build PH151.3A a donc produit une image structurellement correcte pour la priority dot, mais qui a **involontairement perdu** le case summary.
