## 6. Auth Foundation

- **NextAuth.js** avec CredentialsProvider
- Endpoint : `POST /api/v1/auth/admin/login` (à implémenter côté API)
- **Middleware** : protège toutes les routes sauf `/login`, `/api/auth/`, `/_next/`
- **RBAC-ready** : types `AdminRole` (super_admin, ops_admin, account_manager, support_agent, viewer)
- **Hook `useAuth()`** : user, isAuthenticated, hasRole()
- **Session JWT** : durée 8h

## 7. API Foundation

- **ApiClient** : GET, POST, PUT, PATCH, DELETE
- **Auth token** : injectable via `setToken()`
- **Typed responses** : `ApiResponse<T>`, `PaginatedResponse<T>`
- **Env config** : API URL DEV/PROD séparées, internalUrl pour server-side

## 12. DNS — Action requise

Le DNS `admin-dev.keybuzz.io` a été supprimé en PH86.0 (quarantaine legacy).
Il doit être recréé pour que l'admin v2 soit accessible publiquement.

**Enregistrements A à créer :**
- `admin-dev` → `49.13.42.76`
- `admin-dev` → `138.199.132.240`

**Raison du blocage :** Le token Hetzner DNS API n'est pas disponible sur le bastion. Le token hcloud (serveurs) ne couvre pas les zones DNS.

**Impact :** Le pod fonctionne, le service et l'ingress sont actifs, mais :
- L'URL publique `admin-dev.keybuzz.io` ne résout pas
- Le certificat TLS (cert-manager HTTP-01) ne peut pas être validé

**Solution :** Fournir le token DNS Hetzner ou créer manuellement les enregistrements A.
